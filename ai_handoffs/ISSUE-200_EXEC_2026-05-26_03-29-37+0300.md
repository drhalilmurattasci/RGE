### Q2. Safety invariants

Inspected the interactions between queue scope guard, retry paths, transient recovery, branch/publish flow, and snapshot restore. **No path was found where automation could stage or publish work outside the active TASK surface, retry a semantic failure, auto-recover a non-transient failure, or lose user work.**

Specific invariants observed:

- **No publish outside TASK surface.** `Invoke-QueueScopeGuard` runs at `Invoke-AiDispatchQueue.ps1:1290` immediately before `git add -A` at line 1294 and well before `git checkout main` / merge / push. Failure paths call `Fail` (line 614/620/648), which exits non-zero before any mutation. Disallowed-path output enumerates every offending entry (line 644–654).
- **No retry of semantic failures.** `Invoke-WithMutationRetry` wraps only the `Invoke-ClaudeExecute` invocation, not the subsequent `if ($execResult.status -ne 'executed') { Fail … }` block (`Invoke-AiDispatchLoop.ps1:1466`). Similarly, verification-failure handling, Codex control verdict dispatch, and `MaxCorrectionRounds` exhaustion are outside any retry wrapper.
- **No auto-recovery of non-transient failures.** `Get-RecoveryDecision` (`Invoke-AiDispatchAuto.ps1:255`) requires exactly one open failed autonomous issue and exactly one taxonomy label from `$transientLabels` (line 510 — only `stall` and `timeout`). Multiple failed issues, mixed taxonomy, missing taxonomy, non-transient taxonomy, closed failures, or an existing `ai-dispatch-recovered-transient` marker all fall through to the human-review halt (line 313–322).
- **No user-work loss in mutation retry.** `New-MutationSnapshot` captures tracked staged+unstaged via `git stash create`, untracked non-ignored as in-memory bytes, and pins the stash with `refs/dispatch-snapshot/<label>` so a stray `git gc` cannot reap the snapshot during the retry window (comments at lines 169–173). `Restore-MutationSnapshot` resets, cleans, replays the stash with `--index`, and re-writes the untracked manifest. The pinned ref is dropped in the `finally` of `Invoke-WithMutationRetry` (lines 340–343).
- **Stall watchdog can never become a generic timeout label.** `Get-FailureTaxonomyLabels` checks `codex exec stalled` / `no log growth` (line 1031) before checking `timed out` / `timeout` (line 1034). The ordering comment at line 1020 makes this guarantee explicit.
- **Same-phase retry leaves no partial-attempt artifact masquerading as the retry result.** `CleanupPaths` for plan-gate retry covers `.md` / `.envelope.json` / `.stderr.txt` (line 1412); for control retry covers `.json` / `.log` (line 1535). On exhaustion the original `Fail` wording is re-issued so terminal taxonomy classification is preserved.

**One operational hazard noted but not unsafe**: the scope guard's positive-token parser at `Invoke-AiDispatchQueue.ps1:477` requires **backtick-quoted** path tokens. When the Codex Planner emits an `### MAY edit` list with bare bullets (e.g., `- Invoke-AiDispatchLoop.ps1` instead of `- \`Invoke-AiDispatchLoop.ps1\``) the parser yields zero positive tokens, the guard fails closed at line 619, and a Codex-control-passing dispatch is blocked from publish. This is *correct* fail-closed behavior — no unsafe work is published — but it is the dominant observed throughput hit (see Q3, Q5).
