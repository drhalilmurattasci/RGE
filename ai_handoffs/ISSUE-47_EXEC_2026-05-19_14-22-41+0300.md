## What I Changed

### Source

- `crates/material-graph/src/lib.rs`: Implemented GitHub issue #47 â€” added
  `rge_kernel_graph_foundation::VizAdapter` support to `MaterialGraph`.
  - Widened the existing `rge_kernel_graph_foundation` import to also bring in
    `EdgeView`, `NodeView`, and `VizAdapter` (all re-exported from the crate
    root).
  - Added a private free function `material_edge_label(edge: MaterialEdge) ->
    &'static str` in a new `Graph-viewer adapter` section. It matches
    exhaustively over the 16 `(src_port, dst_port)` `PortType` pairs and
    returns lower-case `src->dst` string literals using the port names
    `scalar`, `vector`, `color`, `texture` (e.g. `scalar->color`,
    `texture->vector`). Every arm is a string literal, so the helper never
    allocates and satisfies the `&'static str` lifetime that
    `EdgeView::label` borrows.
  - Added `impl VizAdapter for MaterialGraph`:
    - `node_count` / `edge_count` delegate straight to
      `self.graph.node_count()` / `self.graph.edge_count()` â€” no duplicate
      count state.
    - `nodes()` maps `Graph::nodes()` `(NodeId, &MaterialNode)` pairs to
      `NodeView { id, display_name: node.key.as_str(), kind: "MaterialNode" }`.
    - `edges()` maps `Graph::edges()` `(EdgeId, &EdgeRecord<MaterialEdge>)`
      pairs to `EdgeView { id, src: record.src, dst: record.dst, label:
      material_edge_label(record.data) }`.
  - No change to `MaterialGraph::new`, `Default`, `add_node`, `connect`,
    `node_count`, `edge_count`, `PortType`, `MaterialEdge`, `MaterialNode`,
    or `material_edge_id`. No fields added/removed; no `PortType`
    variant/discriminant changes.
