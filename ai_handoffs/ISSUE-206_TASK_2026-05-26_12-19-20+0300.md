## Planner Notes

Initial planner audit found this expected path: `Invoke-AiDispatchAuto.ps1`
documents `PublishMode branch` as human-gated branch mode and appends
`-NoPublish` to Queue only when `$PublishMode -eq 'branch'`; it appends
`-TraceTiming` independently. `Invoke-AiDispatchQueue.ps1` exposes
`[switch]$NoPublish`, computes `$eligibleForPublish` from successful commit,
loop exit 0, and Codex `pass`, and then skips the publish block when
`$NoPublish` is set. ISSUE-202 and ISSUE-204 trace files both show Auto
starting in `mode=branch` and Queue skipping publish with
`NoPublish=true, eligibleForPublish=true`; their dispatch logs/control JSON
show successful control-passed work ready for publish. This points to
intended documented behavior, not a publish bug, but the Executor must
re-check the evidence and report the final classification.
