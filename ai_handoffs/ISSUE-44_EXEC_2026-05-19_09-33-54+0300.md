### Tests

- `crates/cad-projection/tests/face_picking_smoke.rs::pick_punches_through_topology_changing_boolean_root`:
  the single new `#[test]`. It builds a real `OperatorNode::Boolean(BooleanOp::union())`
  graph root from two distinct Cuboid inputs — a 4×4×4 Cuboid on Boolean
  port 0 and a dimensionally-perturbed 4×4×4 Cuboid (`depth = 4.0001`, so a
  distinct content-derived `NodeId`) routed through a `TransformOp`
  (`translation = [1.0, 1.0, 1.0]`) into the overlap zone on port 1. The
  Boolean node is spawned as its own entity with
  `BRepHandle.brep_owner = Some(ENTITY_OWNER)`. A separate plain 1×1×1
  Cuboid entity is committed in the same graph with
  `BRepHandle.brep_owner = Some(SECOND_OWNER)`.

  The test asserts the invariant directly: the Boolean entity projects a
  non-empty mesh (`triangle_count() > 0`) yet `brep_face_id_for_triangle`
  returns `None` for every Boolean-root triangle — Boolean is
  topology-changing for the B-Rep face resolver. A single ray from
  `[0.0, 0.0, 5.0]` along `[0.0, 0.0, -1.0]` geometrically hits the front
  Boolean union solid first (its solid straddles the z-axis up to z≈3,
  hit near `t ≈ 2.0`), then the behind Cuboid's `PosZ` face at `t = 4.5`.
  `pick_face` returns the behind Cuboid: `pick.entity == behind_entity`,
  `pick.owner == SECOND_OWNER`,
  `pick.face_id == BRepFaceId::for_cuboid_face(SECOND_OWNER, CuboidFaceTag::PosZ)`,
  `pick.t ≈ 4.5` — proving the owned-but-unresolvable front Boolean hits
  are walked past, not masked.
