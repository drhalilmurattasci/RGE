## What I Changed

### Source

- `crates/editor-shell/src/lifecycle/mod.rs`: imported `CursorGrabMode`, added
  a lifecycle-local cursor-grab helper that uses the existing optional winit
  `Window`, warn-logs `set_cursor_grab` failures non-fatally, and no-ops when
  no window exists.
- `crates/editor-shell/src/lifecycle/mod.rs`: kept the existing viewport and
  finite-cursor drag gates, then attempts `CursorGrabMode::Confined` before the
  orbit or pan drag becomes active.
- `crates/editor-shell/src/lifecycle/mod.rs`: releases with
  `CursorGrabMode::None` only after the released drag was active and no viewport
  orbit or pan drag remains active.
- `crates/editor-shell/src/lifecycle/mod.rs`: added test-only fake-window
  instrumentation to record grab/release attempts and simulate grab failure
  without constructing an OS window.
- `crates/editor-shell/src/lifecycle/viewport_navigation.rs`: made the existing
  drag `is_active` helpers available to production code so release ordering can
  check both viewport drag states.

### Tests

- `crates/editor-shell/src/lifecycle/tests.rs::viewport_right_button_orbit_starts_only_with_cursor_and_viewport_hit`:
  now proves orbit start attempts a grab only after the existing cursor and
  viewport gates pass.
- `crates/editor-shell/src/lifecycle/tests.rs::viewport_right_button_orbit_release_stops_future_cursor_rotation`:
  now proves single active orbit release emits the cursor-grab release after
  stopping the drag.
- `crates/editor-shell/src/lifecycle/tests.rs::viewport_middle_button_pan_starts_only_with_finite_cursor_and_viewport_hit`:
  now proves pan start follows the same finite cursor, viewport, and grab policy.
- `crates/editor-shell/src/lifecycle/tests.rs::viewport_middle_button_pan_release_stops_future_cursor_translation`:
  now proves single active pan release emits the cursor-grab release after
  stopping the drag.
- `crates/editor-shell/src/lifecycle/tests.rs::viewport_drag_cursor_grab_no_window_and_failed_grab_preserve_drag_activation`:
  proves no-window shells still activate drags without grab events, and a
  simulated grab failure does not block pan activation.
- `crates/editor-shell/src/lifecycle/tests.rs::viewport_drag_cursor_grab_release_waits_until_all_viewport_drags_stop`:
  proves right-first and middle-first release order does not release the cursor
  while the other viewport drag remains active.
