## Goal

Add one explicit no-shortcut Edit-menu command,
`Command::DeleteCurrentCadCuboid`, that exposes the already-audited headless
`EditorShell::delete_current_cad_cuboid()` path without broadening generic
Delete, Cut, wrapper-world deletion, editor-actions, CAD crates, Cargo metadata,
or egui-host rendering/projection logic. The menu and command palette must show
"Delete Current CAD Cuboid"; it must be enabled only when the current selection
is exactly the single shell-tracked CAD entity; and routing must warn and swallow
rejection without fallback, selection clearing, or bus-stack growth. As the final
implementation step, append exactly one gated source/docs-read-only audit task
167 to `.ai/dispatch.tasks.md` using the self-re-arm block below.

## Deliverables

- `editor-ui` has a new core `Command::DeleteCurrentCadCuboid` variant with
  stable diagnostic id `delete_current_cad_cuboid`.
- `PredicateContext` has one new bool,
  `has_current_cad_cuboid_selection`, defaulting to false.
- `default_editor_menu()` registers one standalone Edit entry with id
  `edit.delete_current_cad_cuboid`, label `"Delete Current CAD Cuboid"`,
  command `Command::DeleteCurrentCadCuboid`, no shortcut, and enablement
  `c.is_editing && c.has_current_cad_cuboid_selection`.
- `EditorShell::predicate_context()` publishes
  `has_current_cad_cuboid_selection` by calling the same crate-internal helper
  used by routing.
- `EditorShell::route_menu_command(Command::DeleteCurrentCadCuboid)` uses the
  exact tracked-CAD selection helper as its route guard, calls
  `delete_current_cad_cuboid()` only when that guard passes, and logs/swallows
  any false guard or entry-point rejection with no fallback to
  `delete_selected_entities`, no selection clearing, and no bus-stack growth.
- Existing `Command::Delete`, the Delete-key accelerator, Cut, and
  wrapper-world delete behavior remain unchanged from the current source.
- `editor-egui-host` changes are limited to expected-list fixture updates in
  `src/menu_tests.rs` and `src/shortcut_help.rs`; no host rendering/projection
  logic changes.
- `.ai/dispatch.tasks.md` appends exactly one bounded source/docs-read-only
  task 167 audit block and no task 168.

## Acceptance Criteria

- The Edit menu includes `"Delete Current CAD Cuboid"` in the registered Edit
  order with no keyboard shortcut or passive shortcut hint.
- The command palette includes the same item as `"Edit: Delete Current CAD
  Cuboid"` through the existing generic projection.
- The new item is enabled only when all are true: Editing state, `self.cad_entity
  == Some(e)`, entity selection cardinality is exactly 1, and the selected entity
  is `e`. Empty, non-CAD single, mixed, stale/None-tracked, and PIE states disable
  the item.
- The enablement predicate and the route guard share one rule by reusing
  `EditorShell::delete_menu_selection_is_exact_tracked_cad_entity` for both;
  the helper may be changed from private to `pub(crate)` only.
- `route_menu_command(Command::DeleteCurrentCadCuboid)` deletes the current CAD
  cuboid through `delete_current_cad_cuboid()`, advances the bus on success,
  supports undo/redo with a fresh restored entity, and prunes CAD entity/face
  selections on success.
- Rejected/stale dispatch of `Command::DeleteCurrentCadCuboid` is warn-and-swallow
  only: no fallback, no wrapper-world deletion, no selection clear, no face
  selection prune, no bus cursor movement, and no stack growth.
- `Command::Delete` still resolves from `"edit.delete"` with the `Delete` key and
  keeps its current behavior. `Command::Cut` and `cut_selected_entities` remain
  wrapper-world only.
- `crates/editor-shell/src/lifecycle/commands.rs`, `editor-actions`, CAD crates,
  Cargo files, and egui-host rendering/projection logic are unchanged.
- All verification gates listed below pass, and `git diff --name-only` is confined
  to the allowed files plus generated current-dispatch artifacts.
