### Tests

- `crates/cad-projection/tests/render_adapter_smoke.rs::fillet_render_mesh_face_labels_align_with_projection_lookup`:
  new additive `#[test]` (Test 5) covering `CadProjection::render_mesh_for`
  against a projected `CuboidOp -> FilletOp` root. The test:
  - Constructs a deterministic 1×1×1 `CuboidOp`, obtains a real upstream
    Cuboid edge via `cuboid.brep_edge_ids(ENTITY_OWNER)` (no `BRepEdgeId`
    synthesized by hand), and builds
    `FilletOp::new(&cuboid, ENTITY_OWNER, vec![edges[0]], 0.1)`.
  - Builds and commits a `CadGraph` where the Cuboid node feeds the Fillet
    node and the graph root is the Fillet node.
  - Spawns + projects the entity on the **Fillet root node** (not the
    upstream Cuboid node) and sets `BRepHandle.brep_owner = Some(ENTITY_OWNER)`
    before projection/lookup assertions.
  - Calls `projection.tick`, then both `projection.projected_mesh(entity)` and
    `projection.render_mesh_for(entity, &world)` on the Fillet-bound entity.
  - Asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64` label per
    projected triangle, and equals the projected mesh's `TopologyFaceId.0`
    labels by triangle index (value + order, not merely `Some(_)`).
  - For every non-degenerate renderer label, wraps the `u64` as
    `TopologyFaceId(label)`, resolves it through
    `brep_face_ids_for_node(graph.graph(), fillet_node, ENTITY_OWNER)`, and
    asserts the resolved `BRepFaceId` equals
    `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
    for the same triangle.
  - For every `TopologyFaceId::DEGENERATE` renderer label, explicitly asserts
    picker-side `brep_face_id_for_triangle` returns `None` — chamfer-cap
    geometry is never resolved as an upstream Cuboid face.
  - Asserts at least one inherited non-degenerate label and at least one
    degenerate chamfer-cap label are observed (fixture sanity).
- Updated the file-level header: added a FilletOp (GitHub issue #32) bullet to
  the "verifies" list and a `fillet_render_mesh_face_labels_align_with_projection_lookup`
  entry to the test inventory; added `FilletOp` to the `rge_cad_core` import.
