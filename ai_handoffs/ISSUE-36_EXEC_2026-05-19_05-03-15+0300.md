### Tests

- `crates/cad-projection/tests/render_adapter_smoke.rs::revolve_render_mesh_face_labels_align_with_projection_lookup`:
  new additive `#[test]` (Test 9) for GitHub issue #36. It builds a graph whose
  root is a direct `RevolveOp` node — the canonical Partial-mode square ring
  profile `Polygon2D::new(vec![[1.0, 0.0], [2.0, 0.0], [2.0, 1.0], [1.0, 1.0]])`,
  `segments = 8`, `angle = PI` — via `RevolveOp::partial`, spawns the projected
  entity for that Revolve node, sets `BRepHandle.brep_owner = Some(ENTITY_OWNER)`
  (the existing fixed file-level `const BRepOwnerId`), and ticks the projection.
  It then pins:
  - `ProjectedMesh.triangle_count() == 68` (Partial: `2*n*segments + 2*(n-2)`
    `= 2*4*8 + 2*2 = 64 + 4`).
  - `RenderMesh.face_labels` is `Some` with `len() == 68`, equal to both
    `projected.triangle_count()` and `ProjectedMesh.face_labels.len()`.
  - An ordered per-triangle comparison: renderer-side `u64` label `N` equals the
    projected `TopologyFaceId.0` at index `N` (not sorted/dedup/set-based).
  - For every triangle, the renderer label is non-degenerate, resolves through
    `brep_face_ids_for_node(graph.graph(), revolve_node, ENTITY_OWNER)`, and the
    resolved `BRepFaceId` equals
    `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
    for that same triangle.
  - Added `use std::f32::consts::PI;` and `RevolveOp` to the existing
    `rge_cad_core` import group — the small local import adjustments the new
    test requires.
