### Tests

- `crates/cad-projection/tests/round_fillet_brep_face_id_lookup_smoke.rs::cuboid_through_round_fillet_root_resolves_upstream_and_degenerate_caps`:
  New cargo-discovered integration smoke. Builds a committed `CadGraph`
  with `CuboidOp -> RoundFilletOp`, sets the RoundFillet node as the graph
  root, spawns an ECS entity bound to that RoundFillet root, seeds
  `BRepHandle.brep_owner` with a deterministic `BRepOwnerId`
  (`[0x24; 16]`), and ticks `CadProjection`. The RoundFillet rounds a
  single Cuboid edge (radius 0.1); the edge ID is selected via
  `CuboidOp::brep_edge_ids(TEST_OWNER)` and the operator is built with
  `RoundFilletOp::new(&cuboid, owner, edges, radius)` ŌĆö no edge IDs are
  synthesized by hand and no private RoundFillet internals are touched.
  The test pins:
  * The projected RoundFillet mesh is labeled (`face_labels.is_some()`),
    one label per triangle.
  * The mesh contains inherited non-degenerate Cuboid labels (each is one
    of the six canonical face indices `TopologyFaceId(0..=5)`) and all six
    Cuboid faces from `cuboid.brep_face_ids(TEST_OWNER)` survive as
    inherited labels.
  * The mesh contains at least one `TopologyFaceId::DEGENERATE` label from
    RoundFillet-added cap/corner geometry.
  * Every non-degenerate triangle resolves through
    `CadProjection::brep_face_id_for_triangle` to the upstream Cuboid
    `BRepFaceId` minted under the same `BRepOwnerId` for that exact
    `TopologyFaceId` ŌĆö matched against the direct
    `CuboidOp::brep_face_ids(TEST_OWNER)` provider output, not merely
    "some face id".
  * Every `TopologyFaceId::DEGENERATE` triangle resolves to `None`.
  * Out-of-bounds triangle lookup resolves to `None`.
  * A cleared `BRepHandle.brep_owner` resolves to `None` even with a
    valid, labeled projected mesh.
