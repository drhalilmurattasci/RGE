## What I Changed

### Source

- `kernel/ecs/src/snapshot.rs`:
  - Added `use rge_kernel_diagnostics::{Diagnostic, DiagnosticSink, FailureClass, Span};`.
  - Added `pub fn restore_from_snapshot_with_diagnostics(&mut self, bytes: &[u8], sink: &mut dyn rge_kernel_diagnostics::DiagnosticSink) -> Result<(), SnapshotError>` — the diagnostics-aware entry point (exact signature from the Deliverables).
  - Refactored the existing restore body into a private `restore_from_snapshot_inner(&mut self, bytes, sink: &mut dyn DiagnosticSink)`. The public compatibility method `restore_from_snapshot(&mut self, bytes: &[u8]) -> Result<(), SnapshotError>` is preserved with its exact signature and now delegates with a no-op `&mut ()` sink, so its prior behavior is unchanged.
  - In the unregistered-component skip branch, the existing `tracing::warn!` is preserved; alongside it the helper now emits one warning `Diagnostic` with `FailureClass::SnapshotRecoverable` whose span carries the skipped component snapshot name via `Span::at_asset(name)` (a structured field), and whose message also names the component. Malformed-input handling (BadMagic / BadVersion / Truncated / Serde, including invalid UTF-8 component names and postcard decode failures) is untouched and still returns `SnapshotError`.
- No change to the wire format, `MAGIC`, `VERSION`, `SnapshotComponent`, or `SnapshotError`.
