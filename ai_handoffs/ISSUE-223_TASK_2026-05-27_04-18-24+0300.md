## Deliverables

- A dedicated `.github/workflows/powershell.yml` workflow running on Windows that installs or imports Pester and PSScriptAnalyzer and runs both gates.
- A Pester test under `tools/dispatch-tests/**` that invokes the production queue audit-log writer from `Invoke-AiDispatchQueue.ps1` using synthetic inputs and a temporary filesystem root.
- The Pester test must assert the complete generated audit log contains no unexpanded PowerShell variable tokens matching the regex `\$[A-Za-z_][A-Za-z0-9_]*`.
- The synthetic audit-log test must include values for dispatch id, issue number/title/url, branch, loop path/text, loop exit code, control verdict, generated run files, marker summary, control JSON, git status, git diff name-status, and git diff stat.
- The test must verify representative expected synthetic values appear in the generated log so the no-token assertion is not satisfied by an empty or skipped log.
- A PSScriptAnalyzer invocation that is used locally and in CI against tracked `*.ps1` and `*.psm1` files from `git ls-files`, with `.ai/dispatch-*` generated scratch excluded.
- Any required PSScriptAnalyzer settings must be narrow and documented by filename or inline workflow comments; do not suppress broad rule classes just to make the gate pass.
- One `EXECUTION_REPORT` packet for ISSUE-223 documenting changed files, test commands, analyzer results, and any analyzer cleanups.
