## Goal

Add the GitHub issue #32 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves inherited Cuboid face labels through
the render adapter when the projected root is a `CuboidOp -> FilletOp` graph.
The smoke must exercise the Fillet root, compare non-degenerate renderer-side
opaque labels against the same B-Rep face identity path used by picker-side
lookup, and handle `TopologyFaceId::DEGENERATE` chamfer-cap labels explicitly.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected
  `CuboidOp -> FilletOp` root.
- The test constructs a deterministic `CuboidOp`, obtains a real upstream
  Cuboid edge through `cuboid.brep_edge_ids(ENTITY_OWNER)`, and constructs
  `FilletOp::new(&cuboid, ENTITY_OWNER, vec![edge_id], 0.1)`.
- The test builds and commits a `CadGraph` where the Cuboid node feeds the
  Fillet node, and the graph root is the Fillet node.
- The projected entity is spawned for the Fillet root node, not the upstream
  Cuboid node, and its `BRepHandle.brep_owner` is set to `Some(ENTITY_OWNER)`
  before projection and lookup assertions.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Fillet-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has one opaque `u64`
  label per projected triangle, and stays aligned by triangle index with the
  projected mesh's `TopologyFaceId` labels.
- For every non-degenerate renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, resolves it through the Fillet-root graph resolver
  path using `brep_face_ids_for_node(graph.graph(), fillet_node, ENTITY_OWNER)`,
  and asserts that B-Rep face ID equals the value returned by
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`.
- For every `TopologyFaceId::DEGENERATE.0` renderer label, the test must not
  try to resolve it as an upstream Cuboid face. It may either skip it with an
  explicit comment or assert that picker-side lookup returns `None` for that
  chamfer-cap triangle.
- The test asserts at least one inherited non-degenerate renderer label is
  observed. It may also assert at least one degenerate chamfer-cap label is
  observed as a fixture sanity check.
- A completed execution report packet for this dispatch.

## Acceptance Criteria

- `cargo test -p rge-cad-projection --test render_adapter_smoke` exits 0.
- The new smoke is distinct from the existing Cuboid, Extrude, and
  RoundFillet render-adapter smokes and specifically exercises
  `OperatorNode::Fillet`.
- No `BRepEdgeId` is synthesized by hand; every edge passed into
  `FilletOp::new` comes from the upstream Cuboid's `BRepEdgeProvider` surface.
- The entity under test is bound to the Fillet root, so the assertion proves
  renderer-label alignment through the projected Fillet output rather than
  through an upstream Cuboid projection.
- The renderer-label assertion compares exact upstream Cuboid `BRepFaceId`
  values for inherited non-degenerate labels. The test must not accept only
  `Some(_)`, any face, any owner, a sorted label set, or a deduplicated label
  set.
- `TopologyFaceId::DEGENERATE.0` chamfer-cap labels are ignored or explicitly
  rejected for upstream-face resolution; they must not be treated as Cuboid
  face labels and must not cause the test to require a stable B-Rep face ID for
  cap geometry.
- The implementation does not change renderer-tier crates, B-Rep identity
  derivation, FilletOp geometry, graph resolver behavior, projection lookup,
  picker behavior, ECS handle semantics, Cargo metadata, or unrelated tests.
- If no real render-adapter bug is exposed, tracked project-file changes are
  limited to `crates/cad-projection/tests/render_adapter_smoke.rs`, plus the
  protocol-required execution report packet.
- If a real render-adapter bug is exposed and fixed, tracked project-file
  changes are limited to `crates/cad-projection/tests/render_adapter_smoke.rs`,
  `crates/cad-projection/src/render_adapter.rs`, and the protocol-required
  execution report packet.

## Planner Notes

Use the existing Cuboid renderer-label test in
`crates/cad-projection/tests/render_adapter_smoke.rs` as the closest assertion
model, and use
`crates/cad-projection/tests/brep_face_id_lookup_smoke.rs` only as read-only
reference for constructing the `CuboidOp -> FilletOp` graph, spawning the
entity on the Fillet root, and distinguishing inherited non-degenerate labels
from `TopologyFaceId::DEGENERATE` chamfer-cap labels.

The intended core assertion is: for each Fillet output triangle, the
renderer-side opaque label is the same `TopologyFaceId.0` value carried by the
projected mesh. If that label is non-degenerate, resolving it through
`brep_face_ids_for_node(graph.graph(), fillet_node, ENTITY_OWNER)` must produce
the same `BRepFaceId` as `CadProjection::brep_face_id_for_triangle` for the
same triangle. If that label is `TopologyFaceId::DEGENERATE.0`, it represents
nameless chamfer-cap geometry and must not be resolved as an upstream Cuboid
face.
