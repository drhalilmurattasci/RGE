## Goal

Wire the `rge-runtime-headless` stub to the existing `rge-data` project/scene
schemas and the `rge-scene-loader` bridge for GitHub issue #177. The binary
must load the first scene referenced by a project manifest, advance the loaded
world exactly one tick, and print stable stdout evidence that the tracked
`golden-projects/simple-scene/.rge-project` fixture loads two entities and
reaches tick 2 after the required single tick advance.

## Deliverables

- `rge-runtime-headless` accepts exactly one positional `<project-path>`
  argument and exposes no optional flags.
- The binary parses `<project-path>` as `rge_data::Project`, resolves the first
  `Project.scenes` entry relative to the project manifest directory, and parses
  that file as `rge_data::Scene`.
- The parsed scene is loaded through `rge_scene_loader::load_scene_into_world`.
- The resulting `World` has `World::advance_tick()` called exactly once after
  scene load and before the success output is printed.
- Successful stdout includes literal, testable evidence of `entity_count=2` and
  `current_tick=2` when run against
  `golden-projects/simple-scene/.rge-project`.
- One integration test invokes the built binary against
  `golden-projects/simple-scene/.rge-project`, asserts successful exit, and
  asserts stdout contains the count and tick evidence.
- Dependency additions in `runtime/runtime-headless/Cargo.toml` are the minimum
  needed to parse RON project/scene data and call the scene loader. Do not add a
  CLI framework that exposes implicit flags such as `--help` or `--version`.

## Acceptance Criteria

- `cargo run -q -p rge-runtime-headless -- golden-projects/simple-scene/.rge-project`
  exits 0 and stdout contains both `entity_count=2` and `current_tick=2`.
- `rge-runtime-headless` rejects zero arguments, more than one argument, and
  flag-shaped invocations such as `--help`, `-h`, and `--version`; those cases
  must not print the success count/tick evidence.
- The runtime resolves the scene path from the manifest parent directory, not

- The integration test uses the compiled `rge-runtime-headless` binary and the

- The implementation does not change `rge-data`, `rge-scene-loader`, ECS,
  component, golden-project, or workspace APIs.

## Planner Notes

Relevant existing APIs are already present: `rge_data::{Project, Scene}` parse
from RON via Serde, `Project.scenes` stores project-root-relative scene paths,
and `rge_scene_loader::load_scene_into_world(&Scene)` returns a populated
`rge_kernel_ecs::World` with `entity_count`, `current_tick`, and
`advance_tick` behavior. Keep this dispatch as a thin binary wiring task and
let the existing loader tests continue to own detailed component assertions.

`World::new()` starts at `current_tick() == 1`; the required single
`World::advance_tick()` after loading therefore makes the success output's
kernel tick value `current_tick=2`. Do not print a separate ad-hoc tick counter
that is decoupled from `World::current_tick()`.

`rge-data` and `rge-scene-loader` are not workspace dependencies today. Add
direct path dependencies in `runtime/runtime-headless/Cargo.toml` only, using
the same local-crate style already used elsewhere in the workspace; do not
promote either crate to `[workspace.dependencies]`.

For the integration test, prefer `std::process::Command::new(env!(
"CARGO_BIN_EXE_rge-runtime-headless"))` so no CLI test helper dependency is
needed. Build the fixture path from `env!("CARGO_MANIFEST_DIR")` joined with
`../../golden-projects/simple-scene/.rge-project` so the test is independent of
the process current directory.
