## Goal

Implement GitHub issue #134: add the smallest Ctrl+2 CommandBus time-scale action in editor-shell. The new binding must map Ctrl+2 to `EditorKeyCommand::SetTimeScaleDoubleSpeed`, and handling that command must call `EditorShell::set_time_scale(2.0)` so the existing `SetTimeScale` action and existing `CommandBus::submit` path provide undo/no-op behavior. This dispatch is intentionally narrow: one command enum variant, one key mapping, one handler arm, and focused tests only.

## Deliverables

- `EditorKeyCommand` gains exactly one new production command variant named `SetTimeScaleDoubleSpeed`.
- `EditorKeyCommand::from_key_press(KeyCode::Digit2, true, false)` returns `Some(EditorKeyCommand::SetTimeScaleDoubleSpeed)`.
- `EditorShell::handle_key_command(EditorKeyCommand::SetTimeScaleDoubleSpeed)` routes by calling `self.set_time_scale(2.0)`.
- The implementation uses the existing `SetTimeScale` action and the existing `EditorShell::set_time_scale` -> `submit_action` -> `CommandBus::submit` path.
- Focused tests cover all required behavior:
  - Digit2 key mapping for Ctrl-without-Shift.
  - Fresh-shell Ctrl+2 submit changes `TimeScale` from 1.0 to 2.0.
  - Undo after Ctrl+2 restores `TimeScale` to 1.0.
  - Repeating Ctrl+2 while already at 2.0 is a no-op: no new stack entry and no second time-scale audit event.
- One `EXECUTION_REPORT` packet documents the exact files changed, verification commands run, results, and any deviations or halt condition.

## Acceptance Criteria

- The only production behavior change is Ctrl+2 double-speed time scale through `EditorKeyCommand::SetTimeScaleDoubleSpeed`.
- No new `Action` trait shape, no new production `Action` implementation, no adapter ledger, no adapter authority layer, and no `CompoundAction` wrapper is introduced.
- `EditorShell::handle_key_command` does not directly mutate `TimeScale`, `World`, resources, audit ledgers, or CommandBus internals for Ctrl+2; it delegates to `set_time_scale(2.0)`.
- The existing no-op behavior in `EditorShell::set_time_scale` remains authoritative for repeat Ctrl+2. A second Ctrl+2 at 2.0 must not submit another action, grow the undo stack, flip state beyond the first submit, or record another `TimeScaleChanged` audit event.
- Existing Ctrl+Z, Ctrl+Y, and Ctrl+S mappings and behavior remain unchanged.
- Ctrl+Shift+2 is unbound unless the current mapping signature makes that impossible without broader input-surface changes; in that case halt with `NEEDS_HUMAN`.
- The tests are placed only in `crates/editor-shell/tests/keyboard_command_bus_round_trip.rs` and/or `crates/editor-shell/tests/time_scale_test.rs`.
- The implementation compiles without touching any forbidden file or adding Cargo dependencies.
- The final diff contains no UI, menu, toolbar, egui, editor binary, Cargo, editor-actions, editor-state, ECS, or architecture-doctrine changes.
