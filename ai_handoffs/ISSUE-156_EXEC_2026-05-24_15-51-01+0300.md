  `rge_data::Project`, resolves the first `Project.scenes` path relative to
  the manifest directory, parses that file as `rge_data::Scene`, and bridges
  it into a fresh `rge_kernel_ecs::World` via a private file-local helper
  `load_scene_into_world` that calls
  `World::spawn_with_id(EntityId::from_ulid(entity.id.0))` exactly once per
  parsed scene entity. The test asserts
  `world.entity_count() == scene.entities.len()`, asserts per-entity
  existence through `World::entity(...)`, captures `current_tick()` /
  `last_tick()`, calls `advance_tick()`, then asserts `current_tick()`
  advanced by one and `last_tick()` equals the prior `current_tick()`. The
  helper is identity-only — it does not inspect or translate components,
  relations, assets, transforms, cameras, lights, or any other typed data.
  Count assertions compare against `scene.entities.len()` rather than a
  hard-coded `1`, so the test stays correct if the fixture grows.
