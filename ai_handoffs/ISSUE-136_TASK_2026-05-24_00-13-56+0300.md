## Goal

Implement GitHub issue #136: add the smallest Ctrl+0 CommandBus time-scale reset in editor-shell. The new binding must map Ctrl+0 to `EditorKeyCommand::ResetTimeScaleDefault`, and handling that command must call `EditorShell::set_time_scale(TimeScale::DEFAULT)` so the existing `SetTimeScale` action and existing `CommandBus::submit` path provide no-op, coalescing, dirty-state, audit, and undo behavior. This dispatch is intentionally narrow: one command enum variant, one key mapping, one handler arm, and focused tests only.

## Deliverables

- `EditorKeyCommand` gains exactly one new production command variant named `ResetTimeScaleDefault`.
- `EditorKeyCommand::from_key_press(KeyCode::Digit0, true, false)` returns `Some(EditorKeyCommand::ResetTimeScaleDefault)`.
- `EditorShell::handle_key_command(EditorKeyCommand::ResetTimeScaleDefault)` routes by calling `self.set_time_scale(TimeScale::DEFAULT)`.
- The implementation uses the existing `SetTimeScale` action and the existing `EditorShell::set_time_scale` -> `submit_action` -> `CommandBus::submit` path.
- Focused tests cover all required behavior:
  - Digit0 key mapping for Ctrl-without-Shift.
  - Fresh-shell Ctrl+0 is a no-op because the shell already starts at `TimeScale::DEFAULT`.
  - Ctrl+2 followed by Ctrl+0 after the CommandBus coalesce window resets `TimeScale` to `TimeScale::DEFAULT`.
  - Undo after that Ctrl+0 reset restores the prior 2.0 time scale.
- One `EXECUTION_REPORT` packet documents the exact files changed, verification commands run, results, and any deviations or halt condition.

## Planner Notes

The relevant current shape is already in `crates/editor-shell/src/lifecycle/commands.rs`: `SetTimeScale` exists, `EditorShell::set_time_scale` captures `from`, clamps `to`, short-circuits no-ops, submits through `CommandBus`, and records the editor-shell time-scale audit event only after a successful non-no-op submit. Ctrl+2 already uses this path through `EditorKeyCommand::SetTimeScaleDoubleSpeed`; Ctrl+0 should add the reset command entry point to that same path, not invent a second time-scale mutation surface.

The reset-after-Ctrl+2 test must account for existing coalescing. If Ctrl+0 is submitted inside the 500 ms coalesce window, it can merge into the prior `SetTimeScale` stack entry and undo will not return to 2.0. The required behavior for this dispatch is specifically Ctrl+2, then wait past the existing coalesce window, then Ctrl+0, then undo back to 2.0. Do not change the coalesce policy to satisfy the test.
