## Goal

Close the user-visible `rge-editor --scene <path>` gap surfaced by ISSUE-247's
audit Q4/Q5. Today, `init_render_state` in
`crates/editor-shell/src/render_path.rs` returns before constructing a winit
window, `SurfaceContext`, `EguiHost`, or `InspectorHandoff` whenever neither a
CAD scene nor a prebuilt render mesh is attached. A world-only scene can be
loaded into `EditorShell`, but it produces no visible editor window. Fix this
by constructing the production window/surface/egui host for world-only launches
and by adding an egui-only frame path that can present the dock and Inspector
chrome without a cuboid render pipeline.

This is a render-init and render-frame ordering change only. The
`editor-egui-host` crate API stays frozen.

## Scope

### MAY edit

- `crates/editor-shell/src/render_path.rs`
  - Split `init_render_state` into two phases.
  - Phase 1 runs unconditionally when production winit `resumed` invokes
    `init_render_state`: construct the winit window, `GfxContext`,
    `SurfaceContext`, `EguiHost`, and `InspectorHandoff`.
  - Phase 2 remains gated by `has_cad_scene || has_prebuilt_mesh`: cuboid
    pipeline setup and render-mesh upload, including the current mesh-side
    assertions near `init_render_state_post_surface`.
  - Add an egui-only branch in `render_frame` that activates when Phase 2 did
    not construct the cuboid pipeline/frame graph. That branch must acquire the
    surface texture, clear the target, publish the inspector handoff, call
    `EguiHost::render`, submit, present, and return `true`.
  - Preserve the existing cuboid path as the active branch whenever Phase 2 ran.
- `crates/editor-shell/src/lifecycle/mod.rs`
  - Edit only if the new world-only `render_frame` path exposes a real
    lifecycle tolerance hole, such as a branch that assumes cuboid pipeline
    state exists after Phase 1-only initialization. If touched, the EXEC report
    must state the exact hole and why this file was required.
- `crates/editor-shell/src/lifecycle/tests.rs`
  - First locate any existing direct test of `init_render_state` behavior.
  - If a direct test exists, update its expected behavior to the new contract:
    empty-world production init creates window/gfx/surface/egui/inspector state
    while leaving cuboid pipeline state uninitialized.
  - If no direct test exists and `init_render_state` is unit-testable, add the
    narrowest feasible test for the new contract.
  - If no direct test exists and `init_render_state` is not unit-testable
    because it requires winit's runtime-only `ActiveEventLoop`, do not add a
    fake-runtime test. Document that outcome explicitly in the EXEC report.

## Planner Notes

`ai_handoffs/ISSUE-247_EXEC_2026-05-28_14-53-12+0300.md`, specifically the Q5
