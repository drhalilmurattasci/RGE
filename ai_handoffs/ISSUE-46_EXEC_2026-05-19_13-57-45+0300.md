## What I Changed

### Source

- `crates/material-graph/src/lib.rs`: normalized line endings from CRLF to
  LF. The on-disk file was checked out with CRLF terminators (a Windows
  `core.autocrlf=true` checkout artifact) while `rustfmt.toml` mandates
  `newline_style = "Unix"`; the canonical gate
  `cargo +nightly fmt --all -- --check` reads the on-disk file and reported
  the incorrect newline style. The change removed all 316 CR bytes
  (`0x0D`) that preceded an LF, leaving 316 LF-terminated lines. File size
  went from 10981 to 10665 bytes — exactly the 316 removed CR bytes. No
  BOM was present and none was added. No Rust token, byte order, public
  API, test, comment, or behavior was touched — the change is purely the
  newline style.

Note: the committed git blob for `crates/material-graph/src/lib.rs` was
already LF (verified with `git cat-file -p HEAD:...` and `git show :...`
— both report 0 CR bytes). The CRLF existed only in the working tree.
After normalization the working copy matches the index/HEAD blob, so
`git diff` reports no content change for this file (see Worktree State).
The correction is therefore a working-tree fix that makes the on-disk
file match both the canonical rustfmt policy and the already-LF git blob.
