## Goal

Route the existing first CAD cuboid add through the editor CommandBus without
expanding CAD, render, UI, or Cargo scope. Widen `rge-editor-actions` only with
a minimal generic action-context contract that preserves object-safe
`Box<dyn Action>`, current World-only behavior, coalescing, audit-ledger,
dirty/save-mark semantics, and the crate's independence from editor-shell and
CAD crates. In `rge-editor-shell`, define the CAD action and CAD context, route
`EditorShell::add_cad_cuboid_to_empty_scene` through `command_bus.submit`, and
prove add, undo, and redo leave graph/projection/world/render inspection state
coherent.

This implements task 160 from `.ai/dispatch.tasks.md` for GitHub issue #417.
The final implementation step must append exactly one gated read-only audit

## Acceptance Criteria

- `rge-editor-actions` exposes a generic action-context contract that is
  independent of editor-shell, CAD, and projection concepts.
- `Box<dyn Action>` remains object-safe, and the undo stack still stores dyn
  actions rather than switching to a parallel typed action stack.
- Existing World-only actions, `CommandBus::submit`, `undo`, `redo`,
  coalescing, audit-ledger projection, diagnostics, `mark_saved`, and
  `is_dirty` behavior continue to pass their existing tests.
- Existing World-only call sites either compile unchanged or through a default
  World-context adapter with no new type annotations required in existing
  tests.
- `EditorShell::add_cad_cuboid_to_empty_scene` calls the bus route for the
  successful add path. The direct shell method may remain the public entry
  point, but its mutation authority must be the bus action.
- Undo after the cuboid add restores the pre-add CAD graph/projection/world
  state: graph restored to `pre_add_head`, the spawned B-Rep entity is gone,
  projection render mesh lookup for that entity is gone, tracked CAD entity is
  cleared or non-live as appropriate, and inspection reports no stale entity or
  mesh.
- Redo after undo re-adds the cuboid through the same action and restores a
  coherent graph/projection/world/tracked-entity inspection state.
- Existing empty-scene rejection behavior remains intact for live tracked CAD
  entities, installed CAD graph/projection/world state, render-only content,
  uploaded meshes, and other partial/non-empty scenes.
- Existing stale-id normalization still runs before the empty-scene guard and
  does not treat render-mesh absence alone as stale tracked-CAD state.
- Apply-time undo metadata is handled without `unsafe`, global/static state, or
  unexplained broad interior mutability. Prefer capturing the undo token before
  submit as immutable action data; alternatively, make the smallest justified
  apply/revert mutability change and cover it with tests.
- `.ai/dispatch.tasks.md` ends with exactly one new task 161 audit block for
  this dispatch. Task 160 must not directly record a `NEEDS_HUMAN_RECORDED`
  marker unless appending task 161 is impossible and the EXEC packet explains
  why.
