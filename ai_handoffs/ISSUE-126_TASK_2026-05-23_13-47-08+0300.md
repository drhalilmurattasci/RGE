## Acceptance Criteria

- `crates/gfx/src/frame_graph/compile.rs` no longer exists as a file, and these
  four files exist: `compile/mod.rs`, `compile/error.rs`, `compile/types.rs`,
  and `compile/algorithm.rs`.
- `crates/gfx/src/frame_graph/mod.rs` and `crates/gfx/src/lib.rs` are unchanged.
  Their existing public re-exports continue to expose `CompileError`,
  `ResourceLifetime`, `AliasingGroup`, and `CompiledFrameGraph` with no public
  API change.
- `compile::compile_passes` remains reachable by the existing
  `FrameGraph::compile` caller as `pub(crate)` without editing
  `crates/gfx/src/frame_graph/mod.rs`.
- `CompileError` still has exactly `Cycle` and `UnwrittenResource(ResourceId)`,
  with the same `Debug`, `Error`, `PartialEq`, and `Eq` derives and the same
  `thiserror` messages.
- `ResourceLifetime` and `AliasingGroup` preserve their public shape, derives,
  impl methods, and behavior.
- `CompiledFrameGraph` preserves its private field set and order:
  `execution_order`, `resource_lifetimes`, `aliasing_groups`, and
  `descriptors`.
- `CompiledFrameGraph` preserves its `Clone`, `Debug`, `PartialEq`, `Eq`,
  `Serialize`, and `Deserialize` derives, and the `descriptors` field retains
  `#[serde(skip)]` exactly.
- `structural_hash` preserves the same BLAKE3 byte stream order exactly:
  prefix `b"CompiledFrameGraph/v1\0"`, execution-order `NodeId` strings each
  followed by `b"\0"`, separator `b"\0lifetimes\0"`, resource id bytes,
  `first_use` as `u32` little-endian with the same overflow fallback,
  `last_use` as `u32` little-endian with the same overflow fallback,
  separator `b"\0groups\0"`, `b"group\0"` before each group, and each group
  member's `ResourceId::as_bytes()` in the same order.
- `compile_passes` preserves algorithm behavior exactly: writers map
  collection, unwritten-resource validation, RAW dependency adjacency,
  self-loop filtering, Kahn topological sort using `BTreeSet` deterministic
  tiebreaks, resource lifetime indexing by execution-order position, greedy
  aliasing assignment in `BTreeMap<ResourceId, _>` order, descriptor cloning,
  and error return behavior.
- No test is added, deleted, renamed, or given a changed expectation. Moving
  existing `compile.rs` unit tests into the new module files is allowed only if
  the test names and bodies remain behavior-equivalent.
- No `SPLIT-EXEMPTION` marker is added, no architecture-lint rule changes, and
  no test expectations or snapshots are changed.
- The final tracked diff contains only the required replacement under
  `crates/gfx/src/frame_graph/compile*`. The only allowed untracked additions
  outside source are this dispatch's own EXEC packet and optional generated
  sidecar.
