## Acceptance Criteria

- `edit.delete_current_cad_cuboid` binds exactly
  `Shortcut::new(Modifiers::CTRL | Modifiers::SHIFT, Key::Delete)`.
- Ctrl+Shift+Delete resolves only to `Command::DeleteCurrentCadCuboid`; no
  other command or wrapper delete path handles that accelerator.
- The accelerator uses the existing generic route:
  `keycode_to_shortcut` -> `enabled_command_for_shortcut` ->
  `route_menu_command(Command::DeleteCurrentCadCuboid)`.
- The delete route still uses the exact-tracked-CAD guard and does not fall
  back to `delete_selected_entities`, selection mutation, face-selection
  pruning, stale-CAD cleanup, or bus-stack growth on rejection.
- `AcceleratorTable::detect_conflicts()` and resolved shortcut conflict
  projections stay empty, and the accelerator table contains exactly 19
  entries.
- Bare Delete still resolves to `Command::Delete`; `Command::Cut`,
  `cut_selected_entities`, wrapper-world `delete_selected_entities`,
  editor-actions, CAD crates, editor-state, Cargo metadata, and
  editor-egui-host production rendering/projection logic remain unchanged.
- Only the five MAY-edit docs/task-brief files may change for task content.
- The task records whether a next bounded FEATURE task is in-policy; if not,
  exactly one `NEEDS_HUMAN_RECORDED: ISO-date - concrete reason` marker plus a
  "Recommendation for human approval" block is recorded.

  by this task's final step.
