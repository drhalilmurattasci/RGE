#### Field-visibility note (acknowledged mechanical adjustment)

`CompiledFrameGraph`'s four fields were file-private under the old
single-file layout (no visibility modifier = private to `compile.rs`).
After the split, `compile_passes` (in `compile/algorithm.rs`) must still
construct `CompiledFrameGraph` (defined in `compile/types.rs`), so the
four fields were widened to `pub(super)`. This is the minimum widening
required to keep the same construction-site pattern (struct-literal in
`compile_passes`) without adding any new method, factory, or
public-facing item.

External (out-of-`compile/`) visibility is **unchanged**: the fields
remain unreachable from anywhere outside `crates/gfx/src/frame_graph/compile/`,
so the public API surface of `CompiledFrameGraph` is identical. No
caller in `crates/gfx/src/frame_graph/mod.rs`, `crates/gfx/src/lib.rs`,
`crates/gfx/tests/**`, `resource_map.rs`, `texture_pool.rs`, or any other
file outside the `compile/` directory can observe a behavioral or
visibility change. The fields are still:

- private to the compile substrate as a unit (no public exposure),
- in the same order (`execution_order`, `resource_lifetimes`,
  `aliasing_groups`, `descriptors`),
- with the same derives, the same `#[serde(skip)]` on `descriptors`,
  and the same serde-empty-on-deserialize behavior for that field.

The TASK packet's Acceptance Criteria phrases this as "CompiledFrameGraph
preserves its private field set and order" and as "Do not change ...
field visibility ... except for mechanical rustdoc link path updates
inside compile/**". The widening to `pub(super)` is interpreted as a
necessary mechanical bridge required by the Planner-mandated file layout
(types in `types.rs`, algorithm in `algorithm.rs`); it does not change
external visibility, does not appear in any public-API path, and does
not change any documented compile invariant. Flagged here for Reviewer
visibility; if the Reviewer prefers a `pub(super) fn new(...)`
constructor in `types.rs` instead, that is a strictly mechanical
alternative.
