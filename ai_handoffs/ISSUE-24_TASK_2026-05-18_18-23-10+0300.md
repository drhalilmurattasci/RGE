## Goal

Add a focused cad-projection integration smoke for GitHub issue #24 that proves
`CadProjection::brep_face_id_for_triangle` resolves a projected
`CuboidOp -> RoundFilletOp` root back to the upstream Cuboid's stable
`BRepFaceId`s for inherited face labels, while `TopologyFaceId::DEGENERATE`
round-fillet cap and corner-patch triangles remain nameless and resolve to
`None`.

## Deliverables

- A new integration test file at
  `crates/cad-projection/tests/round_fillet_brep_face_id_lookup_smoke.rs`.
- A small fixture that builds a committed `CadGraph` with `CuboidOp` feeding a
  `RoundFilletOp`, sets the RoundFillet node as the graph root, spawns an ECS
  entity bound to that RoundFillet root, sets `BRepHandle.brep_owner` to a
  deterministic `BRepOwnerId`, and ticks `CadProjection`.
- Assertions that the projected RoundFillet mesh is labeled, contains inherited
  non-degenerate upstream Cuboid face labels, and contains at least one
  `TopologyFaceId::DEGENERATE` label from RoundFillet-added cap/corner geometry.
- Assertions that every non-degenerate `TopologyFaceId(0..5)` inherited from
  the Cuboid resolves through
  `CadProjection::brep_face_id_for_triangle` to the matching upstream
  `CuboidOp::brep_face_ids(TEST_OWNER)` entry.
- Assertions that every triangle labeled `TopologyFaceId::DEGENERATE` resolves
  to `None`.
- Same-fixture `None` checks for out-of-bounds triangle lookup and for a
  missing `BRepHandle.brep_owner`, if the test structure can keep them concise.

## Acceptance Criteria

- The new test file compiles as a cargo-discovered integration test without
  adding Cargo manifest entries.
- The fixture uses public APIs from `rge-cad-core`, `rge-cad-projection`, and
  `rge-kernel-ecs`; it does not reach into private RoundFillet internals.
- The fixture selects Cuboid edge IDs through `CuboidOp::brep_edge_ids(owner)`
  and constructs `RoundFilletOp::new(&cuboid, owner, edges, radius)` rather than
  synthesizing edge IDs manually.
- The entity is bound to the RoundFillet root node, not the upstream Cuboid
  node, so the smoke specifically exercises graph-recursive face lookup through
  `OperatorNode::RoundFillet`.
- Non-degenerate labels resolve to the upstream Cuboid `BRepFaceId`s minted
  under the same `BRepOwnerId`; no test assertion accepts merely "some face id"
  without checking it against the direct Cuboid provider output.
- `TopologyFaceId::DEGENERATE` labels are explicitly observed in the projected
  mesh and every such triangle returns `None` from
  `brep_face_id_for_triangle`.
- Out-of-bounds and missing-owner lookups return `None` when included.
- No production behavior is changed to make the smoke pass. If a production
  edit is made under the compile-only exception, the EXECUTION_REPORT must name
  the exact compile error and explain why the edit is compile-only.
