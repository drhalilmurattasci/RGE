## Goal

Add one bounded `CadProjection::pick_face` smoke test for GitHub issue #44. The test must prove that a front `BooleanOp::union` root entity, even when it has an owner and produces geometric ray hits, is transparent to face picking because the Boolean root is topology-changing and cannot resolve a `BRepFaceId`; the picker must continue through that hit and return the resolvable Cuboid face behind it on the same ray.

## Planner Notes

Existing `face_picking_smoke.rs` already documents and tests the closest-resolvable rule. This issue should add the missing topology-changing Boolean root case next to the existing transparent-front-hit smokes. A practical fixture is a large front Boolean union, such as two distinct Cuboid inputs connected to a `BooleanOp::union` node and spawned from that Boolean node, plus a smaller plain 1x1x1 Cuboid behind it at the origin. With a ray from `[0.0, 0.0, 5.0]` along `[0.0, 0.0, -1.0]`, the Boolean geometry should be hit first while the behind Cuboid's `PosZ` face is hit later at about `t = 4.5`. The important invariant is that the front Boolean entity has an owner and geometric hits but still resolves to no `BRepFaceId`, so `pick_face` walks past it.
