## Goal

Wire the already-implemented opt-in `-EnablePreflightAudit` switch from the
autonomous driver through the queue runner into `Invoke-AiDispatchLoop.ps1`.
ISSUE-200 confirmed that the Loop switch exists but is currently unreachable
from Auto-driven dispatches because neither Auto nor Queue accepts or forwards
the switch. This dispatch adds only the narrow passthrough while preserving
default behavior when the switch is omitted.

## Deliverables

- Add `[switch]$EnablePreflightAudit` to `Invoke-AiDispatchAuto.ps1`.
- When Auto is invoked with `-EnablePreflightAudit`, append
  `-EnablePreflightAudit` to the `Invoke-AiDispatchQueue.ps1` child-process
  arguments.
- Add `[switch]$EnablePreflightAudit` to `Invoke-AiDispatchQueue.ps1`.
- When Queue is invoked with `-EnablePreflightAudit`, pass
  `-EnablePreflightAudit` to `Invoke-AiDispatchLoop.ps1`.
- Keep the passthrough strictly opt-in. When the switch is omitted from Auto or
  Queue, no preflight audit is requested and the generated child-process
  argument lists remain otherwise equivalent to current behavior.
- Preserve existing `-TraceTiming` behavior: Auto still passes `-TraceTiming`
  to Queue only when set, Queue timing trace behavior remains unchanged, and
  the trace JSONL schema is not edited.
- Leave `Invoke-AiDispatchLoop.ps1` untouched. It already owns the
  `EnablePreflightAudit` implementation.

## Acceptance Criteria

- `Invoke-AiDispatchAuto.ps1` has a new optional switch parameter named
  `EnablePreflightAudit` with no default-on behavior.
- `Invoke-AiDispatchAuto.ps1` forwards `-EnablePreflightAudit` to Queue only
  when `$EnablePreflightAudit` is set. There is no environment-variable
  fallback and no unconditional argument.
- `Invoke-AiDispatchQueue.ps1` has a new optional switch parameter named
  `EnablePreflightAudit` with no default-on behavior.
- `Invoke-AiDispatchQueue.ps1` forwards `-EnablePreflightAudit` to Loop only
  when `$EnablePreflightAudit` is set. There is no environment-variable
  fallback and no unconditional argument.
- Default unset behavior is preserved: Auto and Queue invocations that omit
  `-EnablePreflightAudit` do not request any preflight audit from Loop.
- Existing `-TraceTiming` passthrough and timing trace output behavior are
  preserved. The trace JSONL schema and timing event shape are not changed.
- The implementation does not add a scheduler flag, default-on mode, retry
  route, recovery route, dashboard, docs update, schema update, template
  update, task brief update, or publish behavior change.
- The only tracked source changes are in `Invoke-AiDispatchAuto.ps1` and
  `Invoke-AiDispatchQueue.ps1`; other changes are limited to this dispatch's
  own permitted handoff artifacts and exact queue log artifact.
