## Acceptance Criteria

- The audit reads task 172 evidence from local source, `ai_handoffs/ISSUE-7_*`,
  and the task 173 block in `.ai/dispatch.tasks.md`; it does not call `gh` or
  the network for GitHub queue state.
- `crates/editor-ui/src/menus/keybinding.rs` contains only in-memory keybinding
  target, override, override collection, and diagnostic data. It does not read
  or write disk, settings, environment variables, host state, plugin runtime, or
  Cargo-configured resources.
- `crates/editor-ui/src/menus/registry.rs` keeps `resolve(&PredicateContext)` as
  the default path and delegates or behaves equivalently to
  `resolve_with_keybinding_overrides(..., &KeybindingOverrides::default())` for
  the asserted default fields.
- Overrides apply only to cloned resolved entries for the current resolve and do
  not mutate the registry, `default_editor_menu()`, or later normal resolves.
- Unbinding removes only the executable shortcut from the resolved entry,
  accelerator table, `command_for_shortcut`, and
  `enabled_command_for_shortcut`; it does not remove the visible menu entry,
  command, label, order/section data, enablement result, or passive
  `shortcut_hint`.
- Remapping updates the active executable shortcut for one resolve only.
  Override-induced conflicts remain non-fatal `ResolveResult::conflicts` data,
  and `enabled_command_for_shortcut` suppresses conflicted execution.
- Unknown targets are deterministic diagnostics for extension point / entry id
  pairs unknown before visibility filtering. Known but hidden/disabled entries
  are not reported as unknown.
- The shipped default menu remains unchanged: exactly 19 executable
  accelerators, zero conflicts, and no changed entry ids, command ids, labels,
  order, predicates, enablement predicates, shortcut values, or passive shortcut
  hints.
- No Rust source/tests, Cargo, automation, host/shell/runtime, persistence,
  settings UI, plugin runtime, CAD, workflow, schema, or generated
  non-current-dispatch artifact is edited by this audit.
- No task 174 is appended unless it is the final bounded FEATURE task permitted
  by this task's final step. If no such task is in-policy, exactly one completed
  `NEEDS_HUMAN_RECORDED: <ISO-date> - <reason>` marker plus a "Recommendation
  for human approval" block is recorded.
