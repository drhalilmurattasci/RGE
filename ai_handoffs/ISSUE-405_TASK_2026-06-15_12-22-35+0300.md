## Deliverables

- A bounded `EditorShell` CAD add entry point with an explicit name such as `add_cad_cuboid_to_empty_scene`; it must be callable from headless shell tests and must not be wired into menus, shortcuts, palette routing, UI host code, or the CommandBus.
- The entry point creates or uses an empty/new CAD scene only, then executes this existing-API sequence: `CadGraph::begin_operation` -> `OperatorGraph::add_operator(OperatorNode::Cuboid(CuboidOp::default()))` -> `OperatorGraph::set_root` -> `CadGraph::commit` -> `CadProjection::spawn_brep_entity` if needed -> `CadProjection::tick`.
- The shell's CAD fields are coherent after the add: one committed CAD graph root, one projected `BRepHandle` entity, a `render_mesh_for` mesh, and no stale prebuilt-render mesh path substituted for CAD projection.
- Focused lifecycle tests proving add, renderability/frameability/selectability through existing paths, and restore cleanup via `CadGraph::restore_to(pre_add_head)` plus projection cleanup.
- `.ai/dispatch.tasks.md` self-rearm: append exactly one bounded next-task-source AUDIT task as task 151 that mirrors the recent audit task structure and copies this Self-re-arm requirement, or append exactly one `NEEDS_HUMAN_RECORDED: <ISO-date> - <reason>` line if no bounded in-policy next audit exists.

## Acceptance Criteria

- The add entry point succeeds from `EditorShell::new()` or an equivalent new/empty shell and inserts exactly one `CuboidOp::default()`; tests must assert there is exactly one CAD `BRepHandle` entity and exactly one rendered CAD mesh after projection.
- The implementation halts or returns a clear error for non-empty or partially initialized CAD state rather than replacing an existing graph root or inventing multi-root/composition behavior.
- The cuboid is frameable through the existing `current_scene_bounds` / `reset_camera` path and selectable/frameable through the existing `selected_cad_scene_bounds` path after adding the projected entity to `coord.selection`.
- Restore cleanup is covered: capture `pre_add_head = CadGraph::head()` before the add, add and project the cuboid, call `CadGraph::restore_to(pre_add_head)`, clean projection state with existing projection APIs such as `despawn_brep_entity` for the spawned entity, re-tick or validate as appropriate, and assert no stale entity/mesh remains.
- No global undo, Ctrl+Z, dirty/save-mark, or CommandBus claim is made. Do not add or change any `Action`, `CommandBus`, menu, shortcut, palette, save/load, or UI/host behavior.
- `.ai/dispatch.tasks.md` ends with either exactly one new `151.` audit task or exactly one new `NEEDS_HUMAN_RECORDED: <ISO-date> - <reason>` line for this dispatch's self-rearm.
