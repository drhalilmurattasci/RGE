## Goal

Add one focused cad-core integration smoke for GitHub issue #25 proving that a
committed `CuboidOp -> RoundFilletOp` graph chain preserves the upstream
Cuboid's B-Rep face and edge identity vectors through the public graph
resolvers. The smoke should cover both RoundFillet radius variation and selected
edge-set variation, and should pin that neither variation changes the resolved
Cuboid face IDs or edge IDs.

## Deliverables

- A new cargo-discovered integration test file at
  `crates/cad-core/tests/round_fillet_brep_chain_resolution_smoke.rs`.
- A small test fixture that:
  - creates a deterministic `CuboidOp`;
  - gets upstream face IDs via `BRepProvider::brep_face_ids`;
  - gets upstream edge IDs via `BRepEdgeProvider::brep_edge_ids`;
  - constructs `RoundFilletOp::new(&cuboid, owner, selected_edges, radius)`;
  - builds and commits a `CadGraph` with `OperatorNode::Cuboid(cuboid)` feeding
    `OperatorNode::RoundFillet(round)`;
  - resolves at the RoundFillet node with `brep_face_ids_for_node` and
    `brep_edge_ids_for_node`.
- Assertions that every tested RoundFillet radius and selected-edge variation
  returns face ID and edge ID vectors exactly equal to the direct upstream
  Cuboid vectors.

## Acceptance Criteria

- `cargo test -p rge-cad-core --test round_fillet_brep_chain_resolution_smoke -- --nocapture`
  exits 0.
- The new smoke compiles as an integration test without adding or changing any
  Cargo manifest entry.
- The test uses existing public `rge_cad_core` APIs only. It may use the public
  graph scaffolding needed for an integration test (`CadGraph`,
  `OperatorNode`, `BRepOwnerId`, and ID types), but it must not import private
  modules or RoundFillet internals such as `RoundFilletSpec`,
  `RoundFilletPathSpec`, or `RoundFilletUpstream`.
- The upstream reference vectors are produced directly from the Cuboid:
  `cuboid.brep_face_ids(owner)` mapped to `Vec<BRepFaceId>` in emission order,
  and `cuboid.brep_edge_ids(owner)` as `Vec<BRepEdgeId>` in emission order.
- The resolver assertions compare exact vectors with `assert_eq!`; the test
  must not sort, deduplicate, filter, or otherwise normalize the vectors before
  comparison.
- Radius variation is covered with at least three valid positive radii for the
  same selected edge set.
- Selected-edge variation is covered with at least three valid selections,
  including a single-edge selection, a multi-edge selection, and all 12 Cuboid
  edges.
- For every variation, resolved face IDs from the RoundFillet node are
  byte-identical to the upstream Cuboid face IDs, and resolved edge IDs from the
  RoundFillet node are byte-identical to all upstream Cuboid edge IDs, including
  the selected/filleted edges.
- No production source, Cargo metadata, existing tests, docs, scripts, schemas,
  or repository-governance files are changed.
