## Deliverables

- A public read-only CAD inspection snapshot type, with an API shape such as `CadSceneInspection`, exported from the editor-shell lifecycle module if needed by callers/tests.
- A public headless `EditorShell::cad_scene_inspection(&self) -> CadSceneInspection`-style method that reads current shell state only. It must not expose mutable handles and must not enqueue actions, tick projection, initialize render state, upload meshes, resume the window, mutate selection, mutate camera, or touch GPU resources.
- Snapshot fields that are sufficient to inspect the first-cuboid state: CAD graph/world/projection/tracked-entity presence, graph head/root/node count, live `BRepHandle` entity count, projection `render_mesh_for` mesh count, prebuilt render mesh count, uploaded mesh count as a pure `self.meshes.len()` read, and frameability booleans for current-scene and selected-CAD bounds.
- Focused lifecycle tests that exercise the snapshot across fresh empty shell, successful first cuboid add, rejected non-empty add, and restore/despawn cleanup.
- `.ai/dispatch.tasks.md` self-rearm: append exactly one bounded next-task-source AUDIT task as task 153, or append one source-grounded `NEEDS_HUMAN_RECORDED: ISO-date - reason` line if no bounded in-policy audit exists. The task 153 audit must be source/docs-read-only, require dispatcher-provided GitHub-state snapshot evidence for queue/already-filed-task claims, compare current source before choosing the next feature, include explicit `MAY edit`, `MUST NOT edit`, `Done criteria`, `Verification`, and `Halt conditions` sections, and carry this same Self-rearm final-step requirement.

## Acceptance Criteria

- `EditorShell::cad_scene_inspection` or an equivalently named public method exists, returns a value snapshot, and is callable from headless lifecycle tests without constructing a winit window or any `wgpu` resource.
- The snapshot distinguishes the required scenario states through observable fields and operation results: a fresh shell reports no CAD graph/world/projection/entity and zero renderability counts; a successful first add reports one coherent CAD root/node, one live `BRepHandle`, one `render_mesh_for` mesh, zero prebuilt render meshes, and zero uploaded meshes in a headless shell; a rejected non-empty add returns `CadCuboidAddError::SceneNotEmpty` and leaves the before/after inspection unchanged; restore/despawn cleanup reports no live BRep entity and no stale projection mesh.
- Tests assert one CAD node/root, one `BRepHandle`, one `render_mesh_for` mesh, no prebuilt substitution, and no stale entity/mesh after `CadGraph::restore_to(pre_add_head)` plus `CadProjection::despawn_brep_entity`.
- Uploaded mesh count is implemented only as a pure read of `EditorShell.meshes` / `self.meshes.len()`. The implementation must not initialize render state, upload meshes, resume the window, or edit `render_path.rs` to make this count nonzero.
- No menu item, shortcut, command-palette entry, UI/host route, plugin command, CommandBus `Action`, global undo/redo path, dirty/save-mark behavior, save/load/open/close behavior, camera behavior, viewport behavior, render-path refresh, CAD-core behavior, or cad-projection behavior is added or changed.
- `.ai/dispatch.tasks.md` ends with exactly one new bounded task 153 audit entry carrying the required dispatcher-snapshot evidence rule and copied Self-rearm requirement, or exactly one new source-grounded `NEEDS_HUMAN_RECORDED:` line. No other task is added.
