## Deliverables

- A lifecycle-local cursor-grab/release path for active viewport orbit and pan
  drags, using the existing `EditorShell` winit `Window` when present.
- Non-fatal handling of `Window::set_cursor_grab(...)` failures so drag
  activation keeps the current behavior even when the platform rejects the grab.
- Release logic that calls the winit cursor-grab release when right or middle
  mouse release leaves no viewport drag active.
- Focused unit coverage in `crates/editor-shell/src/lifecycle/tests.rs` for
  orbit start, pan start, failed/no-window grab behavior, and release ordering
  when one or both viewport drags are active.
- Status/task docs marking task 133 complete after implementation, without
  appending task 134.

## Acceptance Criteria

- Right-button orbit drag start still requires the existing finite cursor and
  viewport-hit gates; once those gates pass, it attempts a winit cursor grab
  before or as the drag becomes active.
- Middle-button pan drag start follows the same finite cursor, viewport-hit,
  and cursor-grab policy.
- Cursor grab failure is ignored or warn-logged non-fatally and does not block
  the orbit or pan drag from becoming active.
- Right-button release stops orbit dragging and releases the grab only when no
  viewport drag remains active.
- Middle-button release stops pan dragging and releases the grab only when no
  viewport drag remains active.
- A shell with no winit window, such as unit-test/headless construction, does
  not panic and preserves current orbit/pan behavior without an OS cursor grab.
- Existing wheel zoom, right-button orbit math, middle-button pan math,
  left-click face picking, left-double-click frame-all, View menu camera
  commands, command routing, shortcuts, and viewport hit testing are unchanged
  except for the cursor grab/release side effect.
- Focused tests matching `viewport_right_button_orbit`,
  `viewport_middle_button_pan`, and `viewport_drag_cursor_grab` pass under
  `cargo test -p rge-editor-shell --lib`.
- `.ai/dispatch.tasks.md`, `Status.md`, `HANDOFF.md`, `plans/BASELINE.md`, and
  `change.md` record task 133 as done; task 134 is not appended.
- `git diff --name-only` shows only MAY-edit files plus generated ISSUE-387
  artifacts, and `git diff --check` is clean.
