## Deliverables

- A deterministic default clean-release package-set implementation in `tools/compile-timing.ps1`, optionally factored through `tools/Resolve-CleanReleasePackageSet.ps1`.
- A machine-readable package-set surface that reports at least:
  - set name, such as `DefaultCleanRelease`;
  - included package names;
  - excluded package names;
  - an explicit `rge-tool-wasm-bench` include/exclude decision and rationale.
- A generated clean-release cargo command equivalent to `cargo build --release -p PACKAGE_NAME ...`; it must not use `--workspace` for the default clean-release gate.
- Phase 9 clean-release documentation/status updates in `plans/BASELINE.md`, `Status.md`, `HANDOFF.md`, `change.md`, and, only if touched, `.ai/dispatch.tasks.md`.
- Conditional verify/workflow updates only if the implementation narrows an existing workspace gate. If no verify/workflow gate is narrowed, leave `.ai/dispatch.verify.ps1`, `.github/workflows/tests.yml`, and `.github/workflows/bench.yml` unchanged.

## Acceptance Criteria

- The default clean-release package set excludes these five packages from the default release build: `rge-runtime-wasmtime`, `rge-runtime-wasmtime-engine`, `rge-script-host`, `rge-expr-wasm`, and `rge-script-bench`.
- The Planner decision for `rge-tool-wasm-bench` is to include it in the default clean-release package set unless current metadata proves it now pulls Wasmtime or another excluded scripting package. Rationale: it is wasm-named, but current planning reads show it is a stub wrapper with an empty dependency table; including it keeps the default set "all current non-Wasmtime packages" instead of hiding a package solely by name.
- The package set is resolved against `cargo metadata --format-version 1 --no-deps`; the implementation fails clearly if an included/excluded package name is missing, duplicated, or both included and excluded.
- The clean-release measurement command documented in Phase 9 docs is changed from `cargo build --workspace --release` to the explicit default package-set command.
- Existing invocations of `tools/compile-timing.ps1` remain usable for workspace timing unless the docs explain an intentional, compatible parameter addition. Do not silently remove the ability to run workspace timing.
- If any verify/workflow command is narrowed from full workspace coverage to the default package set, explicit opt-in coverage is added for the excluded wasm packages. `cargo bench -p rge-script-bench --no-run` must remain visibly present in `.ai/dispatch.verify.ps1` and `.github/workflows/bench.yml` or be replaced only by an equally explicit command that still compiles that bench package.
- The implementation does not edit Rust source/tests, Cargo manifests, `Cargo.lock`, dependency features, architecture lints, scheduler config, queue/loop publish semantics, or shared target-cache contents.
- A fresh isolated-target clean-release measurement is optional. If it is run, record the result honestly against the Phase 9 <=120s budget. If it is not run, record the exact command that should be run next and do not claim budget closure.
