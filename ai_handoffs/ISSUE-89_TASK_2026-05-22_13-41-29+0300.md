## Goal

Add one test-only `rge-editor::tests` regression for GitHub issue #89 that covers the parser-failure reload path for an existing GLB file whose contents are malformed. The new coverage must be parallel to `r_key_reload_on_missing_file_preserves_prior_frame`, but distinct from the missing-file path: use wrong-magic bytes such as `b"not a glb"`, prove the malformed reload returns `Err`, prove the prior rendered frame is retained, then overwrite the same path with valid GLB bytes and prove a later reload still succeeds and changes the rendered pixel signature.

Required invariants from the issue:

- MUST retain the prior rendered frame after the malformed reload
- MUST follow the failed reload with a valid reload that proves the hook still works

- The test must overwrite the same temporary path with valid GLB bytes from an existing fixture, preferably `textured_uv_cube.glb`, call `shell.handle_asset_reload()` again through the same attached reload source, and assert the valid reload succeeds and changes the rendered pixel signature.

## Acceptance Criteria

- The diff is test-only: source changes are limited to `editor/rge-editor/src/main.rs` inside `#[cfg(test)] mod tests`, plus the execution report packet.
- Exactly one new regression test is added for this issue. It is content-malformed GLB coverage, not another missing-file test.
- The malformed file exists on disk and contains wrong GLB magic bytes such as `b"not a glb"` before the malformed reload is exercised.
- The test asserts the malformed parser-failure path returns `Err`, preferably by directly invoking `GlbLoaderHook.reload_glb(&malformed_path)` before driving `shell.handle_asset_reload()`.
- Frame 1 is rendered from a known valid starting fixture and has a meaningful on-cube central-row signature, including `cube_pixel_count > 8`.
- After `shell.handle_asset_reload()` reads the malformed bytes, frame 2 is rendered from the same shell and has a meaningful on-cube central-row signature. Frame 2 must match frame 1 within `CUBE_THRESHOLD`; an acceptable implementation is asserting the absolute deltas of the central-row average RGB channels are each `<= CUBE_THRESHOLD`.
- The malformed reload must also preserve the prior qualitative signature, for example the `cube.glb` blue-dominant ordering remains true after frame 2.
- The same temporary path is then overwritten with valid GLB bytes from an existing fixture. A subsequent reload through the existing attached hook must succeed and produce frame 3.
- Frame 3 must change the pixel signature relative to frames 1 and 2. An acceptable implementation is the existing `textured_uv_cube.glb` checkerboard assertion (`red_spread > 50 || blue_spread > 50`) plus a mismatch from the frame-1/frame-2 average RGB signature by more than `CUBE_THRESHOLD` on at least one channel.
- No production behavior, watcher behavior, reload semantics, dependencies, fixtures, or docs are changed.
