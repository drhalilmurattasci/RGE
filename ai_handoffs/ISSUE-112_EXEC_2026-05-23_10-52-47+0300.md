### 2. How does the lint detect IO crates and read metadata?

Evidence from `tools/architecture-lints/src/kernel_isolation.rs` (read
in full, 161 lines) and `tools/architecture-lints/tests/kernel_isolation_test.rs`
(read in full, 393 lines):

- **IO crate detection** — `is_io_crate` (source lines 54-71) flags a
  workspace package as an IO crate via two OR-ed checks:
  1. Package name starts with `rge-io-` (canonical workspace prefix;
     e.g. `rge-io-gltf`) or with bare `io-` (synthetic fixture
     convention used by the integration tests).
  2. Manifest-path fallback: when the manifest lives at
     `crates/io-*/Cargo.toml` (relative to workspace root), the crate
     is detected even if the package name does not match.
  Test 6 (`test_rge_prefixed_io_crates_overlap_detected_via_name_path`,
  test lines 257-299) exercises the name-prefix path with directories
  outside `crates/io-*`; test 8
  (`test_manifest_path_fallback_alone_detects_io_crate`, test lines
  347-392) exercises the manifest-path fallback with package names that
  do not start with `io-` or `rge-io-`.
- **Metadata opt-in shape** — each IO crate opts in by adding to its
  own `Cargo.toml`:
  ```toml
  [package.metadata.rge]
  formats = ["gltf", "glb"]
  ```
  (source doc-comment lines 19-26, mirrored verbatim by
  `pkg_toml_with_formats` in the integration tests, test lines 47-58).
- **String shape** — strings are extension names: lower-case, no
  leading dot (source doc-comment line 28). `extract_formats` (source
  lines 77-87) walks
  `pkg.metadata.get("rge").and_then(|rge| rge.get("formats")).and_then(|f| f.as_array())`
  and calls `str::to_lowercase` on every entry (line 84). No alias
  normalization is performed: `jpg` and `jpeg` are treated as distinct
  literals; the lint only compares lower-cased literal strings.
- **Overlap = violation** — a format extension claimed by ≥ 2 detected
  IO crates is a violation; the lint exits 1 (source lines 128-157;
  tests 4, 5, 6, 8). Duplicate literal strings across IO crates are
  therefore violations regardless of whether they refer to the same
  underlying format family.
- **Missing metadata = stderr warning, not violation** — Option B,
  source doc-comment lines 30-35 and the `None =>` branch at lines
  109-115 print
  `warning: io-* crate `<name>` missing `package.metadata.rge.formats` declaration`
  to stderr and continue; the lint still exits 0 if no overlaps exist.
  This is pinned by test 7
  (`test_missing_metadata_is_warning_not_violation`, test lines
  301-334).
