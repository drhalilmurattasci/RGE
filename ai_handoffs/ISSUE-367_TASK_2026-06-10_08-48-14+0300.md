## Deliverables

- A focused `WindowEvent::MouseWheel` route in `EditorShell::window_event` that checks whether the current pointer is over the Viewport tab body before changing the camera.
- Reuse of the existing zoom helpers: vertical wheel delta greater than zero zooms in via `zoom_camera_in`, vertical wheel delta less than zero zooms out via `zoom_camera_out`, and zero vertical delta is a no-op. Horizontal wheel delta must not drive zoom by itself.
- A small private helper is allowed, either inside `lifecycle/mod.rs` or in the optional `lifecycle/viewport_navigation.rs`, if it keeps delta/sign/gating tests simple and does not expose new public API.
- Focused lifecycle tests covering delta direction, viewport-only gating, no-cursor/no-viewport no-op behavior, and preservation of the existing target/direction zoom invariants.
- Updated task bookkeeping only if needed in `Status.md`, `HANDOFF.md`, and/or `change.md`.
- One execution report packet under `ai_handoffs/` for this dispatch.

## Acceptance Criteria

- `WindowEvent::MouseWheel` over the Viewport tab zooms the editor camera and uses the same eye-target distance changes as the existing View menu zoom helpers.
- Wheel events outside the Viewport tab do not zoom, even if they are otherwise unconsumed by egui.
- A shell with no egui host, no cursor position, or no captured viewport rect treats wheel input as a no-op and leaves the camera unchanged.
- Positive vertical wheel delta routes to zoom-in behavior; negative vertical wheel delta routes to zoom-out behavior; zero vertical delta routes to no-op behavior.
- The target, up vector, FOV, near/far clip planes, and normalized eye-target direction remain preserved according to the existing zoom invariant tests.
- No command enum, command bus, accelerator, menu item, route-replacement, orbit, pan, drag, clipboard, CAD, undo-stack, or dirty-state behavior is added or changed.
- No Cargo metadata or dependency file changes occur.
- No forbidden paths from MUST NOT edit are modified.
