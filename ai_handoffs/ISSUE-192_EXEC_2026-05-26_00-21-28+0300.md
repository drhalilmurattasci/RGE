## What I Changed

### Source

- `Invoke-AiDispatchLoop.ps1`:
  - Introduced a script-scoped `$script:RetryableFailEnabled` flag (default
    `$false`) and updated `Fail` so it `throw`s the message when the flag is
    set, otherwise preserves the existing `[Console]::Error.WriteLine` + `exit
    1` behavior. This lets a same-phase retry catch a Fail from inside the
    eligible read-only model-call path without converting any unrelated Fail
    into a retryable exception.
  - Added `Invoke-WithSamePhaseRetry` helper (one function, ~45 lines)
    that: enables the retry flag inside a `try`/`finally`, runs the supplied
    `[scriptblock]$Action`, catches any failure thrown via the flag, deletes
    the listed `CleanupPaths` so a partial first attempt cannot be mistaken
    for the retry result, and re-runs the action exactly once. It prints
    `[retry] ...` lines via `Write-Host` (information stream — does not
    pollute the output pipeline that callers assign into variables). On
    retry exhaustion the flag is cleared and `Fail` is re-invoked with the
    final attempt's message so the original failure path — including the
    Codex stall watchdog wording — is preserved.
  - Wrapped the Claude plan-gate call site (inside the `for ($i = 0; ...
    MaxPlanRevisions; ...)` loop) with `Invoke-WithSamePhaseRetry`, cleaning
    `claude.plan_gate.rev{N}.md`, `claude.plan_gate.rev{N}.envelope.json`,
    and `claude.plan_gate.rev{N}.stderr.txt` between attempts.
  - Wrapped the Codex control call site (inside the `for ($round = 0; ...
    MaxCorrectionRounds; ...)` loop) with `Invoke-WithSamePhaseRetry`,
    cleaning `codex.control.round{N}.json` and `codex.control.round{N}.log`
    between attempts.

## Open Questions for Reviewer

- The retry wrapper deletes the first-attempt artifacts before retrying
  rather than renaming them to an `.attempt1.*` suffix. The TASK packet
  lists deleting/replacing as the first acceptable approach and asks for a
  "compact implementation"; deletion was chosen for simplicity. If the
  Reviewer prefers preserving first-attempt logs for post-mortem
  visibility, the wrapper can be expanded to rename instead — but that
  would add accumulation across rounds.
- The retry status output uses `Write-Host` rather than `Write-Output` so
  the `[retry] ...` lines do not enter the success-output pipeline (which
  would corrupt callers like `$gate = Invoke-WithSamePhaseRetry ...`).
  This matches how `Fail` writes to `[Console]::Error` and how internal
  helpers avoid `Write-Output`. The Reviewer may want to confirm
  `Write-Host` visibility under the queue runner's log capture.
