## What I Changed

### Source

- `Invoke-AiDispatchLoop.ps1` (line 952, inside the `Invoke-PlanFill`
  planner prompt double-quoted here-string): changed the inner backticks
  wrapping the path token in the worked example from single PowerShell
  backticks to doubled PowerShell backticks. Specifically:
  - Before: `` bullet: ``- `Invoke-AiDispatchLoop.ps1` ``. ``
  - After:  `` bullet: ``- ``Invoke-AiDispatchLoop.ps1`` ``. ``
  In a PowerShell double-quoted here-string, single `` ` `` adjacent to a
  letter or space is consumed as an escape (so the broken form rendered
  as ` `- Invoke-AiDispatchLoop.ps1 ` ` — path NOT backtick-wrapped). The
  doubled `` `` `` pair emits a literal `` ` `` character, so the runtime
  prompt now contains the substring `- ` `Invoke-AiDispatchLoop.ps1` `
  required by the Updated Acceptance Criteria.
