    import to bring in `OpKind`.
  - Added a `VizAdapter` section with two free helper functions:
    - `operator_kind_name(&OperatorNode) -> &'static str` â€” maps the existing
      `OpKind` discriminant to a stable `'static` CAD operator kind name
      (`Boolean`, `Cuboid`, `Extrude`, `Fillet`, `Loft`, `Revolve`,
      `RoundFillet`, `Sweep`, `Transform`). Exhaustive over every variant; no
      debug/RON/parameter formatting, no per-call allocation.
    - `input_port_label(EdgeKind) -> &'static str` â€” maps `EdgeKind::Input(0)`
      and `EdgeKind::Input(1)` to `"input[0]"` / `"input[1]"`, with a
      conservative `'static` `"input[?]"` fallback for any unexpected port.
  - Added `impl VizAdapter for OperatorGraph`: `node_count`/`edge_count`
    delegate directly to the inner `Graph` Tier-A counters; `nodes`/`edges`
    map the inner graph's `nodes()`/`edges()` iterators (deterministic
    substrate `BTreeMap` order) into borrowed `NodeView`/`EdgeView` records.
    `NodeView::display_name` and `NodeView::kind` both use the stable kind
    name (no user-authored node names exist); `EdgeView::label` borrows the
    `'static` port label, so no per-edge allocation.
  - No change to public mutation/evaluation semantics, root handling,
    `inner`/`replace_inner`, counters, `connect`, `add_operator`, id
    derivation, or any operator/`OpKind`/`EdgeKind` definition.
  - Normalized the file's line endings from CRLF to LF â€” see Deviations.
