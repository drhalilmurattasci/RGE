## Deliverables

- Import the existing graph-foundation view types needed by the adapter in `crates/cad-core/src/graph/operator_graph.rs`, such as `VizAdapter`, `NodeView`, and `EdgeView`.
- Add a read-only `impl VizAdapter for OperatorGraph` in `operator_graph.rs`, following the local style used by `MaterialGraph` and `AnimGraph`.
- `VizAdapter::node_count` and `VizAdapter::edge_count` must delegate directly to the inner graph's existing substrate counters.
- `VizAdapter::nodes` must iterate over the existing inner graph nodes in substrate order and return `NodeView` records using the stored `NodeId`.
- Node views must use stable CAD operator kind names derived from the existing `OperatorNode`/`Operator` data. Required current names are `Boolean`, `Cuboid`, `Extrude`, `Fillet`, `Loft`, `Revolve`, `RoundFillet`, `Sweep`, and `Transform`.
- Because `OperatorGraph` has no user-authored node names, `NodeView::display_name` and `NodeView::kind` may both use the stable CAD operator kind name. Do not synthesize parameterized labels or expose serialized operator payloads as names.
- `VizAdapter::edges` must iterate over the existing inner graph edges in substrate order and return `EdgeView` records using the stored `EdgeId`, source node, destination node, and input-port edge label.
- Edge labels must be deterministic labels for existing `EdgeKind::Input(port)` payloads. Current CAD operators use input ports `0` and `1`; label those exact ports clearly, for example `input[0]` and `input[1]`, and provide a conservative static fallback for any unexpected port without changing the graph-foundation trait.
- Add focused unit tests in the existing `#[cfg(test)] mod tests` in the same file. Required test names are:
  - `viz_adapter_counts_observed_through_trait`
  - `viz_adapter_node_views_match_substrate_order`
  - `viz_adapter_edge_views_match_substrate_order`
- The tests must exercise the adapter through `let adapter: &dyn VizAdapter = &g;`, not only through concrete `OperatorGraph` methods.
- The node-view test must assert deterministic substrate ordering by sorting the expected `(NodeId, operator-kind-name)` records by `NodeId` and comparing the observed trait views.
- The edge-view test must assert deterministic substrate ordering by sorting the expected `(EdgeId, src, dst, label)` records by `EdgeId` and comparing the observed trait views.
