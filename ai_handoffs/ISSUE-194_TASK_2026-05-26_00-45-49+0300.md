## Goal

Add `Get-AiDispatchTrends.ps1`, a PowerShell 5.1-compatible, read-only local CLI
that aggregates existing opt-in `.ai/dispatch-trace/*.jsonl` timing events into
plain-text dispatch trend metrics and local alerts. The tool is operator
reporting only: it must consume the current trace files without changing trace
emitters, JSONL schema, dispatch behavior, GitHub state, scheduler behavior, or
any automation policy.

## Deliverables

- Create `Get-AiDispatchTrends.ps1` as a PowerShell 5.1-compatible script with
  no external module dependencies and no network calls.
- Add parameters:
  - `-RepoRoot`, defaulting to the directory containing the script.
  - `-TraceDir`, defaulting to `.ai\dispatch-trace` under `-RepoRoot`; relative
    values resolve under `-RepoRoot`.
  - `-SinceHours`, defaulting to all available parsed events when unset or `0`.
  - `-WarnEmptyCapGapSec`, `-WarnQueueLoopSec`, `-WarnPublishSec`,
    `-WarnGithubFinalizeSec`, and `-WarnInvalidJsonLines`, each with
    conservative defaults.
  - `-FailOnAlert`, which changes only the final exit code after the report is
    printed.
- Read existing JSONL trace files line-by-line. Blank lines may be ignored.
  Malformed JSON lines must be counted by file and in total, reported in the
  output, and must not abort parsing of the remaining lines.
- Build phase duration samples by pairing existing start/done style messages
  within the same trace file and process using current event/message fields;
  use existing `elapsed_seconds` values when available and do not require any
  schema or emitter change.
- Report, when present, samples, average, p50, p95, and max for these spans:
  - `auto tick total`: `auto.tick: start` to `auto.tick: end`
  - `empty-cap gap`: `auto.queue-check: primary done` to
    `auto.cap-check: start`
  - `auto queue invocation`: `auto.tick: queue-invocation start` to
    `auto.tick: queue-invocation done`
  - `queue loop`: `queue.loop: start` to `queue.loop: done`
  - `queue publish block`: `queue.publish: block-entry` to
    `queue.publish: block-exit`
  - `queue GitHub finalize`: `queue.github: comment start` to
    `queue.github: relabel done`
- Keep missing spans non-fatal. If a span has no complete samples, show it as
  absent or zero-sample in the report rather than failing.
- Emit plain text with three clearly labeled blocks named exactly `Summary`,
  `Phase Durations`, and `Alerts`.
- Emit `ALERT` lines for threshold breaches. At minimum:
  - invalid JSON line count exceeds `-WarnInvalidJsonLines`;
  - max `empty-cap gap` exceeds `-WarnEmptyCapGapSec`;
  - max `queue loop` exceeds `-WarnQueueLoopSec`;
  - max `queue publish block` exceeds `-WarnPublishSec`;
  - max `queue GitHub finalize` exceeds `-WarnGithubFinalizeSec`.
- Default behavior must be informational: alerts and malformed JSON lines leave
  exit code `0` unless `-FailOnAlert` is supplied. With `-FailOnAlert`, any
  emitted alert exits non-zero after printing the full report.
- If no trace files or no parseable events exist, print a clear no-data message
  in the normal report shape and exit `0` unless `-FailOnAlert` is paired with
  an alert condition.

## Acceptance Criteria

- The only source file added or changed is `Get-AiDispatchTrends.ps1`.
- The script reads `.ai/dispatch-trace/*.jsonl` through the current event
  fields and does not change trace emitters, event names, JSONL schema, or path
  policy.
- Existing trace files are treated as read-only inputs; the script does not
  write, repair, normalize, truncate, or regenerate trace files.
- Output always contains the `Summary`, `Phase Durations`, and `Alerts` blocks.
- The `Phase Durations` block reports `samples`, `average`, `p50`, `p95`, and
  `max` for each available required span.
- Malformed JSONL lines are counted and reported without aborting the report by
  default.
- `-FailOnAlert` preserves the full report output and then exits non-zero only
  when at least one alert threshold is exceeded.
- Without `-FailOnAlert`, alerts remain informational and the script exits `0`.
- The script handles an empty or missing trace directory gracefully with a
  no-data report and exit `0`.
- The script uses only local filesystem reads plus ordinary console output. It
  performs no GitHub mutations, no network calls, no scheduler changes, no CI
  changes, no retry/recovery behavior changes, and no dispatch automation
  behavior changes.
- No dashboards, scheduled tasks, durable report artifacts, docs, schemas,
  Rust/Cargo changes, Auto runner changes, Queue runner changes, Loop runner
  changes, or failure-taxonomy/recovery-route changes are introduced.

## Planner Notes

The current JSONL trace emitter derives `event` from the text before the first
colon and writes `timestamp`, `elapsed_seconds`, `script`, `pid`, `event`,
`message`, and `repo`. The intended implementation should pair existing
messages, not require new fields. Current examples include:

- `auto.tick: start (...)` and `auto.tick: end (...)`
- `auto.queue-check: primary done (...)` and `auto.cap-check: start`
- `auto.tick: queue-invocation start` and
  `auto.tick: queue-invocation done (...)`
- `queue.loop: start (...)` and `queue.loop: done (...)`
- `queue.publish: block-entry; eligibleForPublish=true` and
  `queue.publish: block-exit (...)`
- `queue.github: comment start` and `queue.github: relabel done (...)`

For percentile math, a deterministic nearest-rank implementation is sufficient
unless a simpler local helper already exists. Prefer compact helpers for
parsing, span pairing, statistics, and alert emission. Keep all failures local
to this read-only report tool.
