## Deliverables

- Extend the shortcut-help row projection in `shortcut_help.rs` so each row can
  expose whether its displayed shortcut is currently conflicted.
- Derive conflict state only from `ProjectedMainMenu.conflicts`, using the
  already-projected shortcut display strings. Do not resolve or inspect
  `MenuRegistry`, editor-ui menu definitions, accelerator tables, plugin
  registries, or any second conflict detector from shortcut help.
- Render the existing Keyboard Shortcuts help State column so conflicted enabled
  rows are visibly distinct from normal enabled rows and ordinary disabled
  rows.
- Keep the help projection read-only: opening, closing, rendering, or building
  help rows must not enqueue menu commands, mutate command-palette state,
  toggle conflict diagnostics, change the projected menu, or affect execution.
- Add or revise focused tests in `shortcut_help.rs` covering conflicted enabled
  rows, normal enabled rows, disabled rows, and read-only no-enqueue behavior.
- Update only the allowed status/task docs if needed to record task 125
  completion and verification results. Do not append task 126.
- Write the EXEC report for this dispatch.

## Acceptance Criteria

- An enabled shortcut-help row whose `shortcut` display matches a
  `ProjectedMainMenu.conflicts[*].shortcut` value exposes and renders a clear
  conflict state in the existing Keyboard Shortcuts help State column.
- An unconflicted enabled row still renders as enabled, and an ordinary disabled
  row still renders as disabled.
- Disabled, enabled, and conflicted state are not collapsed into one boolean in
  the rendered help surface. The implementation may keep `enabled` plus an
  explicit conflict flag, or use an equivalent local state helper, as long as
  tests pin the three visible states and ordinary disabled rows remain visibly
  disabled.
- Conflict annotation is sourced only from the `ProjectedMainMenu` passed to
  `shortcut_help_rows`; no registry re-resolution, hard-coded shortcut table,
  conflict scan over menu definitions, plugin lookup, or new detector is added.
- Existing shortcut-conflict diagnostics in `shortcut_conflicts.rs` keep their
  current behavior and source data.
- Menu click execution, command-palette activation, accelerator execution,
  remapping, persistence, routing, diagnostics, and editor-shell behavior remain
  unchanged.
- `git diff --name-only` contains only MAY-edit files plus this dispatch's
  generated EXEC/run/log artifacts.
- `.ai/dispatch.tasks.md` has no `126.` task heading after the change.
