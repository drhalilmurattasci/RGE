- `crates/io-gltf/src/asset_store_cache.rs` (new): defines
  `pub struct AssetStoreCache` that:
  - Holds a `Box<dyn rge_asset_store::Cache>` backing plus a typed mirror
    `HashMap` per asset family (mesh / material / animation / skeleton /
    image) — required to keep the existing borrowed-`Option<&T>` shape of
    `crate::Cache`'s `get_*` methods unchanged.
  - Implements `crate::Cache` by best-effort write-through to the backing
    plus a typed-mirror insert. The infallible trait contract is preserved
    (insert returns a handle; get returns the typed asset). Backing errors
    in the trait path are intentionally discarded — callers that need to
    observe them use the fallible API.
  - Provides additive fallible adapters
    `try_insert_mesh` / `_material` / `_animation` / `_skeleton` / `_image`
    that return `Result<*Handle, GltfError>` and surface
    `rge_asset_store::CacheError` as `GltfError::Cache` via a new
    `impl From<rge_asset_store::CacheError> for GltfError`. The typed
    mirror is not updated when the backing put fails, so adapter state
    stays consistent on error.
  - Exposes static helpers `asset_id_for_{mesh,material,animation,skeleton,image}`
    that map a typed `*Handle` to the corresponding `rge_asset_store::AssetId`
    via `AssetId::from_raw(h.0)`. Canonical byte forms below are chosen so
    `AssetId::from_bytes(canonical_bytes) == AssetId::from_raw(handle.0)`,
    keeping the asset-store entry and the typed handle byte-for-byte
    identical.
  - Exposes `backing(&self) -> &dyn rge_asset_store::Cache` so tests (and
    callers that need to query / evict the backing directly) can probe
    the seam without breaking encapsulation.
  - Constructors: generic `new<C: Cache + 'static>(backing: C)` for
    concrete impls and `from_box(Box<dyn Cache>)` for dyn-selected impls.
  - Private free functions `mesh_canonical_bytes`,
    `material_canonical_bytes`, `animation_canonical_bytes`,
    `skeleton_canonical_bytes`, `image_canonical_bytes` emit the BLAKE3
    pre-image used by each asset family's existing `content_hash()`
    method. Reusing the same byte stream guarantees AssetId/Handle
    parity.
  - No new file other than this module and the EXEC report. No edits
    to existing call sites in editor / loader / importer / exporter /
    integration-test paths — those continue to use `MemoryCache` by default.

### Tests

All seven tests live in `crates/io-gltf/src/asset_store_cache.rs::tests`:

- `insert_mesh_returns_same_handle_as_memory_cache` — pins parity:
  `MemoryCache::insert_mesh` and `AssetStoreCache::insert_mesh` (backed
  by `rge_asset_store::InMemoryCache`) yield the same `MeshHandle` for
  the same `MeshAsset`, and `get_mesh` returns the original asset.
- `re_inserting_same_mesh_yields_stable_handle` — pins handle stability:
  inserting the same mesh twice produces the same `MeshHandle` and the
  typed mirror's mesh count is exactly 1.
- `duplicate_inserts_yield_one_asset_store_entry` — pins byte-level
  dedup through the asset-store backing: after two identical inserts,
  `adapter.backing().len() == 1`, and `backing().get(asset_id_for_mesh(h))`
  returns `Some(canonical_bytes)`. Asserts dedup is observed on the
  asset-store path, not just the typed mirror.
- `distinct_meshes_get_distinct_asset_ids` — pins content addressing:
  two meshes differing by a single position component produce different
  handles and the backing ends up with exactly 2 entries.
- `try_insert_mesh_surfaces_backing_error_as_gltf_cache` — pins error
  surface: with a `FailingBacking` test double whose
  `put`/`get`/`evict_lru` always return `CacheError::Io`,
  `try_insert_mesh` returns `Err(GltfError::Cache(_))` (not `Schema`,
  not `Io`, not a panic). Confirms the typed mirror is NOT mutated on
  backing failure (`adapter.meshes.len() == 0`).
- `try_insert_image_surfaces_backing_error_as_gltf_cache` — same error
  surface verified on the image family to confirm the From-impl
  applies uniformly across families.
- `cache_error_bad_asset_id_maps_to_gltf_cache_variant` — pins that
  `CacheError::BadAssetId` (not just `Io`) lands on
  `GltfError::Cache`, so `From<CacheError>` doesn't leak into another
  error category.

Notes on judgement calls within scope:

- The TASK says the fallible adapter API surfaces `rge_asset_store::CacheError`
  as `GltfError::Cache`. I implemented that via
  `impl From<rge_asset_store::CacheError> for GltfError` inside
  `asset_store_cache.rs` (the `impl` lives in the adapter module, not on
  the local `Cache` trait). `GltfError::Cache(String)` is the variant
  surface used by both the From impl and the fallible adapter methods.
- The infallible `crate::Cache::insert_*` methods on `AssetStoreCache`
  attempt the backing `put` and discard its `Result` via `drop(...)`.
  This was the only way to exercise the asset-store seam from the
  existing infallible trait shape without panicking or changing the
  trait. Callers that need to observe backing failures use the
  `try_insert_*` family — matching the Planner Note that "Asset-store
  failures cannot be surfaced as `GltfError::Cache` without panics or
  without changing the local `crate::Cache` trait" is a HALT condition,
  and was avoided here by routing failure observation through the
  additive fallible API instead.
- The canonical byte form for each asset family deliberately mirrors
  the byte stream hashed inside that asset's existing `content_hash()`
  method, so `AssetId::from_bytes(canonical_bytes) ==
  AssetId::from_raw(handle.0)`. This avoids introducing a new
  serialization format (e.g. bincode), which would have added an
  unrelated dependency to `Cargo.lock` and tripped the Halt condition
  on lock-file scope.
