### Q1. Current Responsibilities

`crates/gfx/src/frame_graph/compile.rs` (729 lines, of which ~360 are non-test and ~370 are `#[cfg(test)] mod tests`) is the analytical compilation layer for the frame-graph. Its module doc (`compile.rs:1-32`) names the contract: topological sort + resource-lifetime analysis + transient aliasing groups, fed by `BTreeMap<NodeId, PassNode>` + `BTreeMap<ResourceId, ResourceClassDescriptor>` from `FrameGraph::compile` in `mod.rs:234-241`. The file owns five tightly-coupled concerns:

1. **Compile-time errors** — `CompileError { Cycle, UnwrittenResource(ResourceId) }` (`compile.rs:45-57`), the only error vocabulary the algorithm emits. Re-exported through `mod.rs:65` and `crates/gfx/src/lib.rs:84` as part of the public surface; wrapped by `FrameGraphError::Compile` in `mod.rs:82-83` via the `#[from]` derive.
2. **Resource lifetime range** — `ResourceLifetime { first_use, last_use }` with the `overlaps(&self, other)` predicate (`compile.rs:65-79`). Used by the greedy aliasing assignment and by smoke tests as the structural correctness oracle. Re-exported through `mod.rs:65` and `lib.rs:86`.
3. **Aliasing group** — `AliasingGroup(pub Vec<ResourceId>)` plus the `max_descriptor(&self, &BTreeMap<ResourceId, ResourceClassDescriptor>)` helper (`compile.rs:87-111`) that picks the group's largest descriptor for ADR-118 D5 pool-slot governance. Re-exported through `mod.rs:65` and `lib.rs:84`; consumed by `resource_map.rs:91-123` (`build_resource_map`) which calls `max_descriptor(...)` exactly once per group and routes the result into `TexturePool::acquire` / `BufferPool::acquire` (`resource_map.rs:101-119`).
4. **Compiled payload** — `CompiledFrameGraph { execution_order: Vec<NodeId>, resource_lifetimes: BTreeMap<ResourceId, ResourceLifetime>, aliasing_groups: Vec<AliasingGroup>, descriptors: BTreeMap<ResourceId, ResourceClassDescriptor> }` (`compile.rs:129-141`) with its `Serialize, Deserialize` derive (descriptors are `#[serde(skip)]`), accessors (`execution_order`, `resource_lifetime`, `aliasing_groups`, `descriptors`, `descriptor`, `pass_count`), and the deterministic 32-byte BLAKE3 `structural_hash` (`compile.rs:197-225`) over `execution_order ‖ resource_lifetimes ‖ aliasing_groups` with versioning prefix `b"CompiledFrameGraph/v1\0"` and explicit `\0lifetimes\0` / `\0groups\0` domain separators. Re-exported through `mod.rs:65` and `lib.rs:85`.
5. **The compile algorithm itself** — `pub(crate) fn compile_passes(passes, descriptors) -> Result<CompiledFrameGraph, CompileError>` (`compile.rs:245-359`) implementing the six-step pipeline: (1) writers map; (2) read-vs-writers validation; (3) RAW adjacency with self-loop filtering; (4) Kahn topological sort with deterministic `BTreeSet`-ordered tiebreak; (5) lifetime walk over execution order; (6) greedy aliasing assignment over `BTreeMap<ResourceId,_>` iteration order. `compile_passes` is `pub(crate)`; the sole non-test caller is `FrameGraph::compile` in `mod.rs:240`.

The remaining ~370 lines under `#[cfg(test)] mod tests` (`compile.rs:361-729`) drive 18 unit tests pinning all five concerns: empty-graph and single-pass compile; writer-before-reader and three-pass-chain transitivity; cycle and unwritten-resource error paths; lifetime first/last-use and `None` for unknown ids; `ResourceLifetime::overlaps` boundary behavior; aliasing-group combine vs separate; `structural_hash` determinism + sensitivity to added pass + orthogonality to descriptors; self-read-and-write no-self-loop; descriptor round-trip (texture, buffer, mixed); and four `AliasingGroup::max_descriptor` cases (largest texture, largest buffer, empty group, missing descriptors).

`mod.rs` exposes the API as a single re-export bundle (`mod.rs:64-72`): `pub use compile::{AliasingGroup, CompileError, CompiledFrameGraph, ResourceLifetime}`. `FrameGraph::compile` (`mod.rs:234-241`) clones the inner graph's nodes into a `BTreeMap<NodeId, PassNode>` and threads the descriptor sidecar verbatim into `compile::compile_passes`, mapping `CompileError` into `FrameGraphError::Compile` via the `#[from]` derive on the `Compile` variant.
