## What I Changed

### Source

- `crates/anim-graph/Cargo.toml`: added `rge-kernel-graph-foundation = { workspace = true }`
  under `[dependencies]`. No other dependency was added; the rest of the
  manifest is unchanged.
- `crates/anim-graph/src/lib.rs`: replaced the 4-line stub doc-comment with
  the minimal Phase 8 graph-foundation reuse slice, mirroring the
  `crates/material-graph` precedent:
  - `AnimGraphError(pub GraphError)` newtype with `From<GraphError>`,
    `Display`, and `std::error::Error` (with `source()` exposing the wrapped
    `GraphError`).
  - `AnimState { pub key: String }` — data-only state payload with an
    `AnimState::new(&str)` constructor.
  - `AnimTransition { pub trigger: String }` — data-only transition payload
    with an `AnimTransition::new(&str)` constructor.
  - `AnimGraph` wrapping `Graph<AnimState, AnimTransition>`, with `new`,
    `Default`, `add_state`, `add_transition`, `node_count`, `edge_count`.
  - `add_state` derives `NodeId` via `NodeId::from_bytes(key.as_bytes())`
    and returns the inserted id.
  - `add_transition` derives `EdgeId` deterministically from
    `(src, dst, trigger)` via the `anim_transition_id` helper and returns
    the inserted id.
  - `anim_transition_id` helper: concatenates `src.0` (16 LE bytes),
    `dst.0` (16 LE bytes), and the trigger UTF-8 bytes, then hashes with
    `EdgeId::from_bytes`, so identical transitions collide (duplicate
    detection) while transitions differing only in trigger stay distinct.
  - Mutating methods use `?` so the underlying `GraphError`
    (`DuplicateNode` / `DuplicateEdge` / `DanglingEndpoint`) is preserved
    verbatim inside `AnimGraphError` and remains inspectable.
  - No traversal, evaluation, blend tree, runtime, editor, renderer, ECS,
    asset, serialization, snapshot, or diff behavior was added.

### Tests

Inline `#[cfg(test)] mod tests` in `crates/anim-graph/src/lib.rs`:

- `state_ids_are_stable_across_graphs`: the same key (`"idle"`) in two fresh
  `AnimGraph` instances yields the same `NodeId`.
- `distinct_keys_get_distinct_ids`: `"idle"` and `"run"` yield distinct
  `NodeId`s.
- `add_transition_succeeds_and_updates_counts`: a successful transition
  raises `edge_count` to 1 and preserves `node_count` at 2.
- `duplicate_state_is_rejected`: re-adding `"idle"` fails with
  `GraphError::DuplicateNode(_)`.
- `duplicate_transition_is_rejected`: re-adding an identical
  `(src, dst, trigger)` transition fails with `GraphError::DuplicateEdge(_)`
  and leaves `edge_count` at 1.
- `differing_triggers_are_not_duplicates`: same endpoints with triggers
  `"start_run"` vs `"sprint"` produce 2 distinct edges.
- `dangling_endpoint_is_rejected`: a transition to a never-added node fails
  with `GraphError::DanglingEndpoint { .. }` and leaves `edge_count` at 0.
- `empty_graph_has_zero_counts`: `AnimGraph::default()` reports 0 nodes and
  0 edges.
