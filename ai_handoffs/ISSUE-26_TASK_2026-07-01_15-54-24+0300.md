## Deliverables

- Add `KeybindingOverrides::last_for_target(&self, target: &KeybindingTarget)
  -> Option<&KeybindingOverride>`, returning the last raw override whose target
  equals `target`.
- Add `KeybindingOverrides::effective_overrides(&self) -> impl Iterator<Item =
  &KeybindingOverride>`, collapsing duplicate targets with last-wins semantics
  and yielding the winning overrides in the insertion order of those winning raw
  overrides.
- Add `KeybindingOverrides::targets(&self) -> impl Iterator<Item =
  &KeybindingTarget>`, yielding distinct effective targets in the same
  deterministic order as `effective_overrides()`.
- Preserve raw `iter()`, `push`, `remap`, `unbind`, `from_overrides`,
  `From<Vec<KeybindingOverride>>`, and resolve-time raw insertion-order
  application exactly as behavior.
- Add focused tests in `crates/editor-ui/tests/menus_ordering.rs` and/or an
  inline `#[cfg(test)]` module in `crates/editor-ui/src/menus/keybinding.rs`
  covering duplicate-target last-wins, deterministic effective-target order,
  raw `iter()` preservation, and unbind-vs-remap last-wins.
- Perform the required self-re-arm final step in `.ai/dispatch.tasks.md`
  exactly as specified in this packet.

## Acceptance Criteria

- The new helpers are pure read helpers: they do not mutate the override
  collection, registry, menu entries, settings, persistence, host routing, or
  resolve behavior.
- `effective_overrides()` and `targets()` collapse duplicate targets using
  last-wins semantics. For a raw sequence such as `A1, B1, A2`, the effective
  order is `B1, A2`, not first-target order `A2, B1`.
- `iter()` remains the raw append-only insertion-order iterator and still yields
  duplicate targets.
- `last_for_target()` returns `None` for absent targets and the exact final raw
  override for present duplicate targets.
- Tests cover remap replacing unbind and unbind replacing remap for the same
  target.
- Registry behavior is unchanged: `apply_keybinding_overrides` still applies
  the raw `keybinding_overrides.iter()` sequence, and there is no edit to
  `registry.rs`.
- Default menu invariants remain unchanged: the full default resolve still has
  19 executable accelerators and zero conflicts.
- No prohibited surface changes; after completion only MAY-edit files and
  current ISSUE-26 handoff/sidecar artifacts differ.
- `.ai/dispatch.tasks.md` contains exactly one live task-181
  `NEEDS_HUMAN_RECORDED:` selection marker and no appended task 181.
