   `Wait-GitHubActions.ps1:26` and `AI_DISPATCH_AUTOMATION.md:1344` document
   the `-PullRequest 240` plus `-AcceptPrSideCheck CodeQL` style opt-in, but

   `-AcceptPrSideCheck` is not supplied. Acceptance: add side-effect-free

   wait for and accept a matching-SHA successful PR-side `CodeQL` check without
   also passing `CodeQL` in `-WorkflowName`.
2. **Pending StatusContext entries must remain unresolved** - Codex control
   finding: `Wait-GitHubActions.ps1:275-307` treats StatusContext
   `state = PENDING` or `state = EXPECTED` as a completed non-success
   conclusion and returns `fail`, producing exit code 1. Required change:
   classify pending StatusContext states as pending/unresolved so the waiter
   keeps polling, and exits 2 on timeout if they never resolve. Terminal
   failing StatusContext states should still return exit code 1, and successful
   states should still pass only under the existing strict PR-head-SHA and
   explicit-name rules. Acceptance: add classifier tests for StatusContext
   `PENDING` and `EXPECTED`, plus end-to-end waiter tests showing unresolved
   pending StatusContext evidence keeps polling and times out with exit code 2.
