## Goal

Add the GitHub issue #36 cad-projection integration smoke proving that
`CadProjection::render_mesh_for` preserves `RevolveOp` face labels in the same
triangle order used by `ProjectedMesh.face_labels` and picker-side B-Rep face
lookup. The smoke must build and project a direct `RevolveOp` root entity with
a fixed `BRepOwnerId`, then prove every renderer-side opaque face label resolves
through `brep_face_ids_for_node` to the same `BRepFaceId` returned by
`brep_face_id_for_triangle` for that exact triangle.

## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/render_adapter_smoke.rs` covering
  `CadProjection::render_mesh_for` against a projected `RevolveOp` root entity.
- The test constructs a deterministic direct `RevolveOp` fixture, preferably
  the existing square ring profile pattern used by Revolve projection smokes:
  `Polygon2D::new(vec![[1.0, 0.0], [2.0, 0.0], [2.0, 1.0], [1.0, 1.0]])`,
  `segments = 8`, and partial angle `PI` unless the existing render-adapter
  fixture style makes the full-revolution case a better fit.
- The committed `CadGraph` root is the `RevolveOp` node itself. The projected
  entity must be spawned for that Revolve node, not for an upstream or wrapper
  node.
- The entity's `BRepHandle.brep_owner` is set to `Some(ENTITY_OWNER)` or an
  equivalent fixed local `const BRepOwnerId` before projection and lookup
  assertions. The owner must not be random or derived at runtime.
- The test calls `projection.tick`, then calls both
  `projection.projected_mesh(entity)` and
  `projection.render_mesh_for(entity, &world)` on the Revolve-bound entity.
- The test asserts `RenderMesh.face_labels` is `Some`, has exactly one opaque
  `u64` label per projected triangle, and has the same count as
  `ProjectedMesh.face_labels`.
- For every triangle index, the test asserts the renderer label equals the
  projected `TopologyFaceId.0` value at the same index. This must be an
  ordered per-triangle comparison, not a sorted, deduplicated, or set-based
  comparison.
- For every renderer label, the test wraps the opaque `u64` as
  `TopologyFaceId(label)`, resolves it through
  `brep_face_ids_for_node(graph.graph(), revolve_node, owner)`, and asserts
  that resolved `BRepFaceId` equals
  `projection.brep_face_id_for_triangle(entity, tri, &world, graph.graph())`
  for the same triangle.
- If the partial square ring fixture is used, the test should pin the expected
  `68` projected triangles and `68` renderer labels. If a full square ring
  fixture is used instead, pin the corresponding existing Revolve contract
  count of `64` triangles and labels.
- A completed execution report packet for this dispatch.
