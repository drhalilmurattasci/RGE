## Deliverables

- One new additive `#[test]` in
  `crates/cad-projection/tests/face_picking_smoke.rs` covering
  `CadProjection::pick_face` against a projected `CuboidOp -> RoundFilletOp`
  root.
- The test constructs a `CuboidOp`, obtains a real upstream edge through
  `cuboid.brep_edge_ids(ENTITY_OWNER)`, and constructs `RoundFilletOp` via
  `RoundFilletOp::new(&cuboid, ENTITY_OWNER, vec![edges[0]], 0.1)`.
- The test builds and commits a `CadGraph` where the Cuboid node feeds the
  RoundFillet node, and the graph root is the RoundFillet node.
- The projected entity is spawned for the RoundFillet root node, not the
  upstream Cuboid node, and its `BRepHandle.brep_owner` is set to
  `Some(ENTITY_OWNER)` before projection/picking assertions.
- The test calls `CadProjection::pick_face` with a ray that hits an inherited
  Cuboid face in the RoundFillet output, for example the center of the +Z face
  using origin `[0.0, 0.0, 5.0]` and direction `[0.0, 0.0, -1.0]`.
- The test asserts the returned `FacePick` targets the RoundFillet-bound
  entity, carries `ENTITY_OWNER`, and resolves to the exact upstream Cuboid
  `BRepFaceId`, for example
  `BRepFaceId::for_cuboid_face(ENTITY_OWNER, CuboidFaceTag::PosZ)`.
- A completed execution report packet for this dispatch.

## Planner Notes

Use the existing `pick_resolves_filleted_geometry_top_face` test in
`face_picking_smoke.rs` as the closest local shape, but add a separate
RoundFillet-specific smoke instead of replacing or generalizing the Fillet
coverage. Use
`crates/cad-projection/tests/round_fillet_brep_face_id_lookup_smoke.rs` only as
read-only reference for constructing a `CuboidOp -> RoundFilletOp` graph from a
real upstream Cuboid edge ID and spawning the projected entity on the
RoundFillet root.

1x1x1 Cuboid rounded at one real Cuboid edge still picks the inherited Cuboid
+Z face when the entity is bound to the RoundFillet root and
`BRepHandle.brep_owner == Some(ENTITY_OWNER)`.
