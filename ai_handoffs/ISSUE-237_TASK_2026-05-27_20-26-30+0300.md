## Goal

Fix GitHub issue #237 by hardening the queue scope guard's handling of
backtick-quoted leading-dot path tokens and by making failures between the
queue audit-log write and the publish-decision progress comment visible in
captured queue output. This is dispatch-tooling work only. It addresses the
deterministic post-inner-loop stall class seen on #234, where `.gitattributes`
was explicitly allowed in the active TASK packet but rejected by the queue
scope guard's path-token classifier.

## Acceptance Criteria

- `Test-LooksLikePathToken` accepts practical leading-dot path tokens such as
  `.gitattributes`, `.gitignore`, `.env`, `.envrc`, and slash-bearing dotfile
  paths such as `.cargo/config.toml` when they appear as backtick-quoted path
  tokens in TASK scope.
- The leading-dot acceptance is strict: no whitespace, no prose tokens, no
  broad free-text acceptance. Pester coverage must show ordinary prose tokens
  such as `important` and `the` still return false.
- `Get-TaskPositiveAllowedTokens` includes `.gitattributes` when a TASK packet
  lists `.gitattributes` under `### MAY edit`.
- `Invoke-QueueScopeGuard` passes in a temp-git or synthetic Pester fixture
  where `.gitattributes` is modified and the active TASK packet explicitly
  allows `.gitattributes`.
- Any pre-publish failure after `Detailed dispatch log written` and before the
  `publish-decision` progress comment surfaces in captured queue output. This
  must cover generalized failures such as a thrown guard failure, a native
  command error, or another exception in that window, so the queue no longer
  looks like a silent stall.
- Pester tests under `tools/dispatch-tests/**` cover the classifier
  accept/reject cases, the token extraction case, and the scope-guard
  `.gitattributes` regression.
- `AI_DISPATCH_AUTOMATION.md` is unchanged unless concise documentation is
  needed for the post-log failure visibility guarantee.
- The queue publish path for this dispatch is `Invoke-AiDispatchQueue.ps1
  -PublishMode pr`; it must push `ai-dispatch/ISSUE-237` and open a PR on a
  passing run, and must not merge automatically or push `origin/main`.
