## Goal

Add focused integration coverage for GitHub issue #79: `GraphDiff::between`
must report a same-`EdgeId` endpoint change as a changed edge, preserving the
old and new edge records, without producing added/removed edge or node noise.
This closes the current coverage gap beside the existing payload-only
`changed_edges` test.

## Deliverables

- One new integration test in `kernel/graph-foundation/tests/diff_correctness.rs`
  covering the case where the same `EdgeId` exists in both snapshots but an
  endpoint changes.
- If and only if the new test reveals an actual `GraphDiff::between` defect, a
  minimal fix in `kernel/graph-foundation/src/diff.rs`.
- An `EXECUTION_REPORT` that names the test added, summarizes any source change
  if one was required, and records the verification command result.

## Acceptance Criteria

- `kernel/graph-foundation/tests/diff_correctness.rs` contains exactly one new
  test for same-`EdgeId` endpoint change coverage.
- The test constructs two snapshots with identical node sets and an edge with
  the same `EdgeId` in both snapshots, but with at least one changed endpoint.
- The test asserts `diff.changed_edges.len() == 1`, looks up the changed edge by
  that `EdgeId`, and asserts the old and new `src` / `dst` endpoint values.
- The test asserts there is no node or edge noise:
  `added_edges`, `removed_edges`, `added_nodes`, `removed_nodes`, and
  `changed_nodes` are all empty.
- The test asserts `diff.edge_change_count() == 1` and
  `diff.node_change_count() == 0`.
- No graph-foundation source file is edited unless the added test first exposes
  a real `GraphDiff::between` failure.
- `cargo test -p rge-kernel-graph-foundation --test diff_correctness` exits 0.
