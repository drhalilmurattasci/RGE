## Goal

Wire the existing stale tracked-CAD cleanup helper in lifecycle commands into
the first-cuboid add preflight. `EditorShell::add_cad_cuboid_to_empty_scene`
should call `EditorShell::clear_stale_tracked_cad_entity` before the current
`cad_entity` empty-scene guard, so a stale tracked-id-only residue can be
normalized without broadening the first-cuboid boundary. The accepted behavior
is narrow: stale-id-only shells may add the first cuboid, while live tracked CAD
entities and any installed CAD graph, CAD projection, CAD world, render-only
content, uploaded render state, or other partial scene still reject as
`CadCuboidAddError::SceneNotEmpty`.

## Acceptance Criteria

- A fresh shell still adds exactly one default cuboid through the current
  graph/world/projection/render-frame path.
- A shell whose only non-empty CAD-related state is `self.cad_entity = Some(...)`
  for an id that is not live in an installed CAD world is normalized by
  `clear_stale_tracked_cad_entity()` and may add the first cuboid.
- A shell with a live tracked CAD entity carrying `BRepHandle` still rejects as
  `CadCuboidAddError::SceneNotEmpty`.
- Any installed CAD graph, CAD projection, CAD world, render-only content,
  uploaded mesh state, or other partial/non-empty scene still rejects as
  `CadCuboidAddError::SceneNotEmpty`.
- Cleanup remains liveness-only: render-mesh absence alone is not a stale
  predicate, and cleanup clears only `self.cad_entity`.
- `.ai/dispatch.tasks.md` gains exactly one task 159 audit block and no task
  160, no feature follow-up task, and no extra `NEEDS_HUMAN_RECORDED` marker
  during this implementation dispatch.
