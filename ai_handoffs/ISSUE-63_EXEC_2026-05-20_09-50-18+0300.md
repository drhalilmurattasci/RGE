  real `MaterialGraph`, `AnimGraph`, and `OperatorGraph`. Pins, per domain:

    kind `MaterialNode` on all three; edge endpoints by stable

    `AnimState`; transition endpoints by stable `AnimGraph::add_transition`
    id; label `start_run`.
  - Operator: 3 nodes / 2 edges; two distinct `CuboidOp` payloads (so
    content-derived `NodeId`s do not collide) feeding one `BooleanOp::union`
    on ports 0/1; node display names + kinds `Cuboid`/`Cuboid`/`Boolean`;
    edge endpoints by stable `OperatorGraph::connect` id; labels `input[0]`
    and `input[1]`.
  Lookups are by stable `NodeId`/`EdgeId` returned from the construction
  APIs, not by iteration order — adapter iteration is BTreeMap-sorted, so
  positional indices would be fragile.
