## What I Changed

### Source

- `crates/cad-core/src/topology/face_tag.rs`: Added the `SweepFaceTag`
  `#[non_exhaustive]` enum — `FirstCap`, `LastCap`, and
  `Side { segment_index, edge_index, profile_count, path_segment_count }` —
  with frozen discriminants `FirstCap = 0`, `LastCap = 1`, `Side = 2` in
  `SweepOp::evaluate`'s canonical emission order. Caps are categorical (no
  inner data). The `Side` variant encodes the two topology axes
  (`profile_count` AND `path_segment_count`) so neither change is silently
  preserved. Appended a paragraph to the module doc describing the fifth
  per-operator face-tag enum.
- `crates/cad-core/src/topology/face_id/mod.rs`: Registered `mod sweep;`
  and added the `KIND_SWEEP = b"sweep:"` operator-kind separator constant,
  distinct from the four prior kind separators.
- `crates/cad-core/src/topology/face_id/sweep.rs`: New file. Added
  `BRepFaceId::for_sweep_face(owner, tag)`, BLAKE3 over
  `DOMAIN || owner || b"sweep:" || discriminant`, plus (for `Side` only)
  `segment_index || edge_index || profile_count || path_segment_count` as
  little-endian `u32`s, truncated to 16 bytes — mirroring the
  Extrude/Loft constructors.
- `crates/cad-core/src/operators/sweep.rs`: Added `impl BRepProvider for
  SweepOp`. Returns `2 + n * (m - 1)` pairs — `TopologyFaceId(0)` →
  `FirstCap`, `TopologyFaceId(1)` → `LastCap`, then sides
  `TopologyFaceId(2 + segment_index * n + edge_index)` in segment-major,
  profile-edge-major order, exactly matching the existing `evaluate`
  face-label contract. No change to geometry, validation,
  `structural_hash`, triangle emission, or face-label order.
- `crates/cad-core/src/topology/resolve.rs`: Added the direct
  `OperatorNode::Sweep(op) => Ok(op.brep_face_ids(owner))` arm to
  `brep_face_ids_for_node` so Sweep resolves as a direct face provider.
  Boolean stays in the topology-changing catch-all; no Transform behavior
  added. Updated the module doc, `BRepResolveError::TopologyChangingOperator`
  doc, `brep_face_ids_for_node` doc, and the catch-all comment so they no
  longer describe Sweep as topology-changing.
- `crates/cad-core/src/topology/mod.rs` / `crates/cad-core/src/lib.rs`:
  Exported `SweepFaceTag` through `topology::mod` and the crate root;
  refreshed the module doc bullet list and v0-scope note.
- `crates/cad-core/src/topology/provider.rs`: No content change — the file
  was line-ending-normalized to LF by `cargo fmt` (`git diff` is empty).
