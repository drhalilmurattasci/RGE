## What I Changed

### Source

- `Get-AiDispatchTrends.ps1` (new): PowerShell 5.1-compatible read-only CLI
  that aggregates `.ai/dispatch-trace/*.jsonl` events into a Summary /
  Phase Durations / Alerts plain-text report. Parameters:
  `-RepoRoot`, `-TraceDir`, `-SinceHours`, `-WarnEmptyCapGapSec` (30),
  `-WarnQueueLoopSec` (1800), `-WarnPublishSec` (60),
  `-WarnGithubFinalizeSec` (30), `-WarnInvalidJsonLines` (0),
  `-FailOnAlert`.
  - Walks each JSONL file line-by-line, parses with `ConvertFrom-Json`, and
    tallies malformed lines per file plus in total without aborting.
  - Builds duration samples by FIFO-pairing start/done messages within the
    same trace file (one PID per file), using existing `elapsed_seconds`
    when present and falling back to timestamp deltas. Pairs:
    - `auto tick total`: `auto.tick: start` → `auto.tick: end`
    - `empty-cap gap`: `auto.queue-check: primary done` → `auto.cap-check: start`
    - `auto queue invocation`: `auto.tick: queue-invocation start` → `auto.tick: queue-invocation done`
    - `queue loop`: `queue.loop: start` → `queue.loop: done`
    - `queue publish block`: `queue.publish: block-entry` → `queue.publish: block-exit`
    - `queue GitHub finalize`: `queue.github: comment start` → `queue.github: relabel done`
  - Reports `samples`, `average`, `p50`, `p95`, `max` per span; missing
    spans render as `samples=  0  (no complete pairs)`.
  - Percentiles use deterministic nearest-rank on a sorted `[double[]]`.
  - Emits `ALERT:` lines for: invalid JSON line count over threshold, and
    max-of-span over threshold for empty-cap gap, queue loop, queue publish
    block, queue GitHub finalize.
  - Default exit code is `0` even when alerts fire and even with malformed
    lines. `-FailOnAlert` prints the full report and then `exit 1` iff at
    least one alert was emitted.
  - Empty/missing trace dir yields the same three-block no-data report and
    exits `0`.
