## Deliverables

- An editor-shell confirmation seam, mockable in unit tests, that returns only discard/cancel for dirty close-family requests. It may live in `lifecycle/mod.rs` or the optional `lifecycle/unsaved_changes.rs`.
- File Close guarded as follows: clean Close performs the existing `replace_world(KernelWorld::new())` reset; dirty Cancel or dirty no-hook leaves the world, `SaveSource`, selection, clipboard, dirty state, and pending quit flag unchanged; dirty Discard performs the existing reset path.
- File Quit guarded as follows: clean Quit sets the pending quit request as it does today; dirty Cancel or dirty no-hook leaves all document state and the pending quit flag unchanged; dirty Discard sets the pending quit request without closing or resetting the document.
- Window `CloseRequested` guarded as follows: clean close exits as it does today; dirty Cancel or dirty no-hook does not call `ActiveEventLoop::exit`; dirty Discard calls the existing exit path.
- `editor/rge-editor/src/main.rs` supplies the native `rfd` dialog hook in every launch branch that already attaches the Open/Save dialog hooks, without adding dependencies.
- Focused editor-shell tests whose names include `unsaved`, covering clean and dirty Close, Quit, and window-close cancel/discard behavior, including non-mutation on dirty Cancel and dirty no-hook.
- Standard execution report packet for ISSUE-360 and, only if genuinely useful, forward bookkeeping updates in the allowed status/change files.

## Acceptance Criteria

- Dirty state is determined by `EditorShell::command_bus().is_dirty()`. Clean Close/Quit/window close must not invoke the confirmation hook.
- The confirmation hook receives enough save/source context to word a native dialog around the current document/source, but it must not save, mutate the world, mark saved, or change dirty policy.
- Dirty no-hook is treated as Cancel for all three guarded paths.
- Dirty Cancel leaves document state unchanged: wrapper world, kernel world, `SaveSource`, selection, face selection, shell-local clipboard, command-bus dirty state, and pending quit flag must be unchanged.
- Dirty Discard preserves the existing behavior after the decision: Close resets through the current close/reset path, Quit requests app exit through the current pending-quit path, and window close calls the same event-loop exit path.
- The implementation does not add Save-before-close, auto-save, a Save button in the confirmation flow, command variants, menu routes, accelerator changes, Cargo changes, or changes to editor-ui/editor-egui-host/editor-actions.
- `rge-editor` uses its existing `rfd` dependency for the native dialog; `rge-editor-shell` remains free of `rfd`.

## Planner Notes

Prefer a small enum such as discard/cancel plus a trait or callback owned by editor-shell. The hook should be queried only after `command_bus().is_dirty()` returns true. Pass source/display context derived from `SaveSource` when useful, but treat it as dialog copy only.

For window close, keep the actual `event_loop.exit()` call in the `WindowEvent::CloseRequested` arm, but factor the decision so tests can cover cancel/discard without constructing a native dialog. For Quit, make sure the dirty-cancel path does not set or consume `quit_requested`; tests can use the existing single-consume pending-quit accessor from within the lifecycle test module.
